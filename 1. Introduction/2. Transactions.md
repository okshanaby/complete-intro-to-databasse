## Transactions — Simple Summary

A **transaction** is a group of one or more database queries that are treated as a **single unit of work**.

This ensures that **either every query succeeds or none of them do**. If something goes wrong during the transaction, the database **rolls back** (undoes) all the changes, preventing partial updates.

Transactions are closely related to **Atomicity**, which is the "A" in **ACID**.

## Key Points

* A transaction groups multiple queries together.
* All queries inside a transaction succeed together or fail together.
* If one query fails, all previous changes in the transaction are undone (rolled back).
* While a transaction is running, other queries cannot interfere with its operations.
* Transactions help keep data accurate and consistent.

## Why Transactions Matter

Imagine transferring **$100** from Alice to Bob.

Without a transaction:

```text
Subtract $100 from Alice
(Server crashes)
Add $100 to Bob
```

Result:

* Alice loses $100.
* Bob never receives it.
* The database is left in an incorrect state.

With a transaction:

```text
Begin Transaction
Subtract $100 from Alice
Add $100 to Bob
Commit Transaction
```

If everything succeeds:

```text
Both changes are permanently saved.
```

If any step fails:

```text
Rollback Transaction
```

Result:

```text
Neither account is changed.
```

The transfer either happens completely or not at all.

## Common SQL Transaction Commands

```sql
BEGIN;

UPDATE accounts
SET balance = balance - 100
WHERE id = 1;

UPDATE accounts
SET balance = balance + 100
WHERE id = 2;

COMMIT;
```

If an error occurs:

```sql
BEGIN;

UPDATE accounts
SET balance = balance - 100
WHERE id = 1;

-- Error happens here

ROLLBACK;
```

## Relationship Between Transactions and ACID

Transactions mainly provide **Atomicity**, but they also support other ACID properties:

* **Atomicity** → All operations succeed or fail together.
* **Consistency** → Keeps the database in a valid state.
* **Isolation** → Prevents other transactions from interfering while it runs.
* **Durability** → Once committed, the changes are permanently saved.

## One-Sentence Summary

**A transaction groups multiple database operations into one unit so they either all succeed together or all fail together, ensuring the database remains accurate and consistent.**

# Commands Used

## `BEGIN`

Starts a new transaction.

```sql
BEGIN;
```

### Explanation

* `BEGIN` → Starts a transaction. All following queries become part of the same transaction until `COMMIT` or `ROLLBACK` is executed.

---

## `COMMIT`

Saves all changes made during the transaction.

```sql
COMMIT;
```

### Explanation

* `COMMIT` → Permanently applies all successful changes made within the current transaction.

---

## `ROLLBACK`

Cancels a transaction and undoes all changes made since `BEGIN`.

```sql
ROLLBACK;
```

### Explanation

* `ROLLBACK` → Reverts every change made during the current transaction if an error occurs or the transaction should not be completed.

## Command Flags

No command-line flags (such as `-a`, `--help`, or `-v`) are used in these notes. The SQL transaction commands also do not use flags.
