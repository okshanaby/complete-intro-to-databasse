# RAG Retrieval with Ollama and PostgreSQL — Simple Summary

**RAG (Retrieval-Augmented Generation)** combines **vector search** with an LLM to generate answers using your own data.

Before an AI answers a question:

1. Convert the user's question into a vector (embedding).
2. Search PostgreSQL for the most similar vectors.
3. Retrieve the matching records.
4. Send those records as context to the LLM.
5. The LLM generates a more accurate response.

This example shows how to generate embeddings using **Ollama** (locally) or **OpenAI**, store them in PostgreSQL using the **pgvector** extension, and perform semantic searches. 

## Key Points

* Embeddings are numerical representations of text.
* Use an embedding model (not a chat model) to generate embeddings.
* Store embeddings in PostgreSQL using the `vector` data type.
* Search vectors using the `<=>` similarity operator.
* Every new or updated record should also update its embedding.
* Ollama allows embedding generation locally without API costs.
* OpenAI and other providers can also generate embeddings.

---

# RAG Workflow

A complete RAG application works like this:

```text
User Question
      │
      ▼
Embedding Model
(Convert question to vector)
      │
      ▼
PostgreSQL (pgvector)
(Search similar vectors)
      │
      ▼
Retrieve Similar Records
      │
      ▼
LLM
      │
      ▼
Final Answer
```

Instead of the AI relying only on its own knowledge, it uses information stored in your database.

---

# Embedding Models

An **embedding model** converts text into vectors.

Example:

```text
"Never push to prod on Friday"

          │

          ▼

[0.009, 0.044, -0.143, ...]
```

These numbers represent the meaning of the sentence.

Humans don't interpret them—the database compares them mathematically.

# Enable pgvector

Install the PostgreSQL extension.

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

This adds PostgreSQL support for storing vectors.

---

# Add an Embedding Column

```sql
ALTER TABLE comments
ADD COLUMN embedding vector(768);
```

or

```sql
ALTER TABLE comments
ADD COLUMN embedding vector(1536);
```

The number depends on the embedding model.

Examples:

| Model                  | Vector Size |
| ---------------------- | ----------: |
| nomic-embed-text       |         768 |
| text-embedding-3-small |        1536 |

Every embedding stored in the column must have the same length.

# Embedding Script

```js
import pg from "pg";

const CONNECTION_STRING =
  "postgresql://postgres:mysecretpassword@localhost:5432"; // NOTE you may need to add your database name (e.g. /message_boards) if you aren't using the default postgres database

const PROVIDERS = {
  openai: {
    dimensions: 1536,
    model: "text-embedding-3-small",
    async embed(texts) {
      const res = await fetch("https://api.openai.com/v1/embeddings", {
        method: "POST",
        headers: {
          Authorization: `Bearer ${process.env.OPENAI_API_KEY}`,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          input: texts,
          model: "text-embedding-3-small",
        }),
      });

      if (!res.ok) {
        const err = await res.text();
        throw new Error(`OpenAI API error: ${res.status} ${err}`);
      }

      const data = await res.json();
      return data.data.map((d) => d.embedding);
    },
  },

  ollama: {
    dimensions: 768,
    model: "nomic-embed-text",
    async embed(texts) {
      const res = await fetch("http://localhost:11434/api/embed", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          model: "nomic-embed-text",
          input: texts,
        }),
      });

      if (!res.ok) {
        const err = await res.text();
        throw new Error(`Ollama API error: ${res.status} ${err}`);
      }

      const data = await res.json();
      return data.embeddings;
    },
  },
};

const [mode, providerName, ...rest] = process.argv.slice(2);

if (!PROVIDERS[providerName] || !["generate", "get"].includes(mode)) {
  console.error("Usage:");
  console.error(
    "  node embeddings.js generate <openai|ollama>     Embed all comments",
  );
  console.error(
    '  node embeddings.js get <openai|ollama> "text"    Get a single embedding',
  );
  process.exit(1);
}

const provider = PROVIDERS[providerName];

if (mode === "get") {
  const text = rest.join(" ");
  if (!text) {
    console.error(
      'Provide text to embed: node embeddings.js get ollama "your text here"',
    );
    process.exit(1);
  }

  const [embedding] = await provider.embed([text]);
  console.log(`[${embedding.join(",")}]`);
} else {
  console.log(
    `Using ${providerName} (${provider.model}, ${provider.dimensions} dimensions)`,
  );

  const client = new pg.Client({ connectionString: CONNECTION_STRING });
  await client.connect();

  await client.query("CREATE EXTENSION IF NOT EXISTS vector");

  // Drop and recreate column to match provider dimensions
  await client.query(`
    DO $$
    BEGIN
      IF EXISTS (
        SELECT 1 FROM information_schema.columns
        WHERE table_name = 'comments' AND column_name = 'embedding'
      ) THEN
        ALTER TABLE comments DROP COLUMN embedding;
      END IF;
    END $$
  `);
  await client.query(
    `ALTER TABLE comments ADD COLUMN embedding vector(${provider.dimensions})`,
  );

  const { rows } = await client.query(
    "SELECT comment_id, comment FROM comments ORDER BY comment_id",
  );
  console.log(`Found ${rows.length} comments to embed`);

  const BATCH_SIZE = 50;
  for (let i = 0; i < rows.length; i += BATCH_SIZE) {
    const batch = rows.slice(i, i + BATCH_SIZE);
    const texts = batch.map((r) => r.comment);
    const embeddings = await provider.embed(texts);

    for (let j = 0; j < batch.length; j++) {
      const vectorStr = `[${embeddings[j].join(",")}]`;
      await client.query(
        "UPDATE comments SET embedding = $1 WHERE comment_id = $2",
        [vectorStr, batch[j].comment_id],
      );
    }

    console.log(
      `Embedded ${Math.min(i + BATCH_SIZE, rows.length)}/${rows.length}`,
    );
  }

  console.log("Done!");
  await client.end();
}
```

The provided Node.js script automates the entire process.

It:

1. Connects to PostgreSQL.
2. Enables the `vector` extension.
3. Adds the embedding column.
4. Reads every comment.
5. Sends comments to the embedding model.
6. Saves embeddings into PostgreSQL.

Instead of embedding comments one at a time, it processes them in batches.

# Searching with Vectors

To perform semantic search:

```sql
SELECT
comment,
embedding <=> '[your_vector]'
AS distance
FROM comments
ORDER BY distance
LIMIT 15;
```

The `<=>` operator calculates the distance between vectors.

Smaller distance means greater similarity.

---

# Search Process

```text
Search Text
      │
      ▼
Embedding Model
      │
      ▼
Search Vector
      │
      ▼
PostgreSQL
      │
      ▼
Most Similar Comments
```

# Finding Similar Comments

If a comment already has an embedding, you don't need to generate another one.

```sql
SELECT
comment,
embedding <=> (
    SELECT embedding
    FROM comments
    WHERE comment_id = 1
) AS distance
FROM comments
WHERE comment_id != 1
ORDER BY distance
LIMIT 15;
```

This compares every comment against comment **1**.

The result is the most semantically similar comments.

---

# Why This Is Better Than LIKE

Traditional search:

```sql
WHERE comment LIKE '%deploy%'
```

Only finds comments containing the exact word.

Vector search can find related ideas.

Example:

```text
Never deploy on Friday.
```

and

```text
Weekend releases always cause problems.
```

These share meaning even without sharing many keywords.

Vector search finds both.

# Best Practices

* Generate embeddings for all existing records.
* Generate embeddings for every new record.
* Update embeddings whenever text changes.
* Use the same embedding model consistently.
* Store vectors inside PostgreSQL using pgvector.
* Use semantic search before sending context to an LLM.
