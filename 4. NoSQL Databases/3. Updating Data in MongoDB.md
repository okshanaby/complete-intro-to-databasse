# Updating Data in MongoDB — Simple Summary

MongoDB provides several methods to **insert, update, replace, and delete documents**.

For most operations, MongoDB offers methods for handling either **one document** or **multiple documents**.

Examples:

* `insertOne()` / `insertMany()`
* `updateOne()` / `updateMany()`
* `deleteOne()` / `deleteMany()`

Using these specific methods is recommended because they make your intentions clear and help prevent mistakes.

## Key Points

* Prefer `insertOne()` and `insertMany()` over `insert()`.
* Prefer `updateOne()` and `updateMany()` over `update()`.
* Prefer `deleteOne()` and `deleteMany()` over `delete()`.
* Updates require:

  * A filter (which documents to update)
  * An update operation (what to change)
* MongoDB supports **upsert**, which updates a document if it exists or inserts a new one if it doesn't.
* `replaceOne()` replaces an entire document.
* `findOneAnd...` methods combine finding and modifying into one atomic operation.
* `bulkWrite()` executes many write operations in a single request.

---

# `insertOne()` vs `insertMany()` vs `insert()`

MongoDB previously had an `insert()` method.

It behaved differently depending on what you passed to it:

* Object → Insert one document
* Array → Insert many documents

Example:

```javascript id="9ls4sv"
db.pets.insert({
  name: "Luna"
});
```

or

```javascript id="l4z72l"
db.pets.insert([
  {
    name: "Luna"
  },
  {
    name: "Fido"
  }
]);
```

Because this behavior could lead to mistakes, `insert()` has been **deprecated**.

Instead, use the explicit methods:

```javascript id="kfjlwm"
db.pets.insertOne({
  name: "Luna"
});
```

```javascript id="rtj9lo"
db.pets.insertMany([
  {
    name: "Luna"
  },
  {
    name: "Fido"
  }
]);
```

These methods clearly indicate whether you're inserting one or many documents.

# Updating Documents

Updating requires **two things**:

1. A filter (which documents to update)
2. An update operation (what should change)

Example:

```javascript id="jlwmto"
db.pets.updateOne(
  {
    type: "dog",
    name: "Luna",
    breed: "Havanese"
  },
  {
    $set: {
      owner: "Brian Holt"
    }
  }
);
```

The first object searches for the document.

The second object describes the update.

---

# `$set`

`$set` changes or adds field values.

Example:

```javascript id="yb0vv7"
{
  $set: {
    owner: "Brian Holt"
  }
}
```

If the `owner` field doesn't exist, MongoDB creates it.

If it already exists, MongoDB updates it.

---

# Updating Multiple Documents

Use `updateMany()` to modify all matching documents.

Example:

```javascript id="3j3jvf"
db.pets.updateMany(
  {
    type: "dog"
  },
  {
    $inc: {
      age: 1
    }
  }
);
```

This increases every dog's age by 1.

---

# `$inc`

`$inc` increments (or decrements) numeric values.

Example:

```javascript id="5owt0l"
{
  $inc: {
    age: 1
  }
}
```

If `age` is 5:

```text id="ycgsbh"
Before: 5
After: 6
```

---

# `replaceOne()`

Unlike `updateOne()`, `replaceOne()` replaces the **entire document**.

Example:

Original document:

```json id="w9kc8m"
{
  "name": "Luna",
  "type": "dog",
  "age": 10,
  "owner": "Brian"
}
```

Replacement:

```javascript id="r6zz0c"
db.pets.replaceOne(
  {
    name: "Luna"
  },
  {
    name: "Luna",
    age: 11
  }
);
```

Result:

```json id="8fqkiz"
{
  "name": "Luna",
  "age": 11
}
```

Notice that:

* `type`
* `owner`

are removed because they weren't included in the replacement document.

---

# Upsert

**Upsert** combines **Update** and **Insert**.

If a matching document exists:

* Update it.

Otherwise:

* Insert a new document.

Example:

```javascript id="e4hvrj"
db.pets.updateOne(
  {
    type: "dog",
    name: "Sudo",
    breed: "Wheaten"
  },
  {
    $set: {
      type: "dog",
      name: "Sudo",
      breed: "Wheaten",
      age: 5,
      index: 10000,
      owner: "Sarah Drasner"
    }
  },
  {
    upsert: true
  }
);
```

If `Sudo` exists:

* It is updated.

If not:

* MongoDB inserts a new document.

## Best Practice

When using upsert, provide a **complete document**.

Otherwise, newly inserted documents may be missing important fields.

---

# Deleting Documents

Deleting works similarly to querying.

Delete one document:

```javascript id="j3qadw"
db.pets.deleteOne({
  name: "Luna"
});
```

Delete multiple documents:

```javascript id="nz79l2"
db.pets.deleteMany({
  type: "reptile",
  breed: "Havanese"
});
```

All matching documents are removed.

---

# `findOneAnd...` Methods

Sometimes you need to:

* Find a document
* Update/Delete/Replace it
* Return the original document

MongoDB provides three methods:

* `findOneAndUpdate()`
* `findOneAndReplace()`
* `findOneAndDelete()`

These perform everything in **one atomic operation**.

This prevents **race conditions**, where another process changes the document between separate queries.

By default, these methods return the document **before** it was modified.

---

# Race Condition

A race condition occurs when multiple operations try to modify the same data simultaneously.

Bad approach:

```text id="0kpzb0"
Find document

Another user changes it

Update document
```

The data may no longer be what you expected.

Using `findOneAndUpdate()` performs both actions together, avoiding this problem.

---

# `bulkWrite()`

Sometimes an application needs to perform many database operations.

Instead of sending each operation individually:

```text id="2mth9r"
Insert

Wait

Update

Wait

Delete

Wait
```

MongoDB can execute them together.

Example structure:

```javascript id="fcwnm9"
db.pets.bulkWrite([
  {
    insertOne: {
      document: {
        name: "Luna"
      }
    }
  },
  {
    updateOne: {
      filter: {
        name: "Luna"
      },
      update: {
        $set: {
          owner: "Brian"
        }
      }
    }
  }
]);
```

`bulkWrite()` executes each operation in the order provided.

Benefits:

* Fewer network requests.
* Better performance.
* Cleaner application code.

# Common Update Operators

| Operator | Purpose                              |
| -------- | ------------------------------------ |
| `$set`   | Set or replace a field's value       |
| `$inc`   | Increase or decrease a numeric value |

MongoDB supports many more update operators, but these are among the most commonly used.

# One-Sentence Summary

**MongoDB provides specialized methods for inserting, updating, replacing, deleting, and upserting documents, allowing you to modify one or many documents efficiently while supporting atomic operations and bulk updates.**
