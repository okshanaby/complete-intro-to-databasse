# Querying MongoDB — Simple Summary

Once data is stored in MongoDB, you use **queries** to retrieve it.

A query can:

* Find one document
* Find many documents
* Count documents
* Filter documents
* Sort results
* Limit results
* Return only selected fields (projections)

MongoDB queries use **JavaScript objects** instead of SQL statements.

## Key Points

* `findOne()` returns one matching document.
* `find()` returns all matching documents.
* MongoDB returns results using a **cursor** (iterator) by default.
* `countDocuments()` counts matching documents.
* `limit()` limits the number of returned documents.
* `toArray()` converts query results into an array.
* Query operators (`$gt`, `$lt`, etc.) allow comparisons.
* Logical operators (`$and`, `$or`, etc.) combine conditions.
* `sort()` orders query results.
* Projections choose which fields are returned.

---

# Finding One Document

Use `findOne()` when you only need one matching document.

Example:

```javascript
db.pets.findOne();
```

Since no query is provided, MongoDB returns the first document it finds.

Example result:

```json
{
  "name": "Luna",
  "type": "dog",
  "breed": "Havanese",
  "age": 14
}
```

---

# Filtering Results

You can search for documents by providing a query object.

Example:

```javascript
db.pets.findOne({
  index: 1337
});
```

This returns the document where `index` equals `1337`.

You can also search using multiple conditions.

```javascript
db.pets.findOne({
  name: "Spot",
  type: "dog"
});
```

MongoDB returns a document only if **both conditions** match.

---

# Finding Multiple Documents

Use `find()` to retrieve all matching documents.

```javascript
db.pets.find({
  type: "dog"
});
```

This returns every document where the type is `"dog"`.

---

# Cursors (Iterators)

MongoDB doesn't return every matching document immediately.

Instead, it returns a **cursor** (also called an **iterator**).

A cursor lets MongoDB send results in small batches instead of loading everything into memory at once.

Example:

```javascript
db.pets.find({
  type: "dog"
});
```

Initially, MongoDB displays about **20 documents**.

To continue reading the next batch:

```javascript
it
```

Each time you run:

```javascript
it
```

MongoDB shows the next set of results until none remain.

Benefits:

* Uses less memory.
* Efficient for very large collections.
* Lets you process large datasets gradually.

---

# Counting Documents

Use `countDocuments()` to count matching documents.

```javascript
db.pets.countDocuments({
  type: "dog"
});
```

Example output:

```text
2500
```

This means there are 2,500 dogs in the collection.

---

# Limiting Results

Sometimes you only want a certain number of documents.

Example:

```javascript
db.pets.find({
  type: "dog"
}).limit(40);
```

Only the first **40 matching documents** are returned.

---

# Converting Results to an Array

To return all query results at once as a JavaScript array:

```javascript
db.pets.find({
  type: "dog"
}).limit(40).toArray();
```

Example output:

```javascript
[
  {
    name: "Luna",
    type: "dog"
  },
  {
    name: "Fido",
    type: "dog"
  }
]
```

---

# Query Operators

MongoDB provides comparison operators for filtering data.

Example:

```javascript
db.pets.countDocuments({
  type: "cat",
  age: {
    $gt: 12
  }
});
```

This counts cats older than 12 years.

## Common Query Operators

| Operator | Meaning                          |
| -------- | -------------------------------- |
| `$gt`    | Greater than (`>`)               |
| `$gte`   | Greater than or equal (`>=`)     |
| `$lt`    | Less than (`<`)                  |
| `$lte`   | Less than or equal (`<=`)        |
| `$eq`    | Equal (`==`)                     |
| `$ne`    | Not equal (`!=`)                 |
| `$in`    | Value exists in an array         |
| `$nin`   | Value does not exist in an array |

---

# Example: Not Equal

Find all pets named **Fido** that are **not dogs**.

```javascript
db.pets.find({
  type: {
    $ne: "dog"
  },
  name: "Fido"
});
```

---

# Logical Operators

Logical operators combine multiple conditions.

Example:

Find birds between **4 and 8 years old**.

```javascript
db.pets.find({
  type: "bird",
  $and: [
    {
      age: {
        $gte: 4
      }
    },
    {
      age: {
        $lte: 8
      }
    }
  ]
});
```

MongoDB also supports:

| Operator | Purpose                             |
| -------- | ----------------------------------- |
| `$and`   | All conditions must be true         |
| `$or`    | At least one condition must be true |
| `$nor`   | None of the conditions may be true  |
| `$not`   | Negates a condition                 |

**Note:** `$not` and `$ne` are different.

* `$not` negates another query condition.
* `$ne` simply means "not equal."

---

# Special Operators

MongoDB also supports special query operators.

## `$type`

Checks a field's data type.

Example use:

* Number
* String
* Object
* Array

---

## `$exists`

Checks whether a field exists.

Example:

```javascript
{
  age: {
    $exists: true
  }
}
```

Returns documents that contain an `age` field.

MongoDB also includes many advanced operators, including geospatial queries for location-based searches.

---

# Sorting Results

Use `sort()` to order query results.

Descending age:

```javascript
db.pets.find({
  type: "dog"
}).sort({
  age: -1
});
```

Ascending age:

```javascript
db.pets.find({
  type: "dog"
}).sort({
  age: 1
});
```

Sort values:

| Value | Meaning    |
| ----- | ---------- |
| `1`   | Ascending  |
| `-1`  | Descending |

---

# Projections

A **projection** controls which fields are returned.

Example:

```javascript
db.pets.find(
  {
    type: "dog"
  },
  {
    name: 1,
    breed: 1
  }
);
```

Only these fields are returned:

* `name`
* `breed`
* `_id` (included automatically)

---

# Excluding `_id`

To hide MongoDB's automatically generated `_id` field:

```javascript
db.pets.find(
  {
    type: "dog"
  },
  {
    name: 1,
    breed: 1,
    _id: 0
  }
);
```

You can also use boolean values.

```javascript
db.pets.find(
  {
    type: "dog"
  },
  {
    name: true,
    breed: true,
    _id: false
  }
);
```

---

# Excluding Fields Instead

Instead of specifying fields to include, you can exclude specific fields.

Example:

```javascript
db.pets.find(
  {
    type: "dog"
  },
  {
    _id: 0
  }
);
```

This returns every field except `_id`.

---

# One-Sentence Summary

**MongoDB queries use JavaScript objects to search, filter, count, sort, and retrieve documents, with powerful operators that make it easy to work with both simple and complex data.**
