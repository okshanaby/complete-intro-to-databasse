# MongoDB Aggregation — Simple Summary

**MongoDB Aggregation** is a feature used to process, transform, and analyze data inside a collection.

It allows you to perform tasks like:

* Counting documents
* Grouping data
* Filtering results
* Sorting results
* Calculating totals and averages
* Finding patterns and insights

MongoDB provides two ways to perform aggregation:

1. **Aggregation Pipelines** (recommended)
2. **Map-Reduce** (older approach)

---

# 1. Map-Reduce

**Map-Reduce** comes from functional programming concepts.

It works using two functions:

* **Map function** → processes each document individually.
* **Reduce function** → combines the processed results into a final result.

Example idea:

```
Documents:

Dog age: 2
Dog age: 5
Dog age: 10

Map:
Convert each document into an age group

Reduce:
Combine groups and count them
```

However, MongoDB aggregation pipelines are usually preferred because they:

* Perform better
* Are easier to read
* Are easier to maintain

---

# 2. Aggregation Pipelines

An **aggregation pipeline** is a sequence of stages.

Each stage:

1. Receives data from the previous stage.
2. Performs an operation.
3. Sends the result to the next stage.

Think of it like a factory production line:

```
Raw Data
   |
   v
Filter Data
   |
   v
Group Data
   |
   v
Sort Results
   |
   v
Final Output
```

Each stage performs one specific task.

# Example Collection

Imagine we have a `pets` collection:

```json
[
  {
    "name": "Max",
    "type": "dog",
    "age": 2
  },
  {
    "name": "Bella",
    "type": "dog",
    "age": 7
  },
  {
    "name": "Rocky",
    "type": "cat",
    "age": 5
  }
]
```

We want to know:

* How many puppies?
* How many adult dogs?
* How many senior dogs?

---

# Using `$bucket` Stage

```javascript
db.pets.aggregate([
  {
    $bucket: {
      groupBy: "$age",
      boundaries: [0, 3, 9, 15],
      default: "very senior",
      output: {
        count: { $sum: 1 },
      },
    },
  },
]);
```

### Result

Example output:

```json
[
  {
    "_id": 0,
    "count": 1
  },
  {
    "_id": 3,
    "count": 1
  },
  {
    "_id": 9,
    "count": 2
  },
  {
    "_id": "very senior",
    "count": 1
  }
]
```

---

## Explanation

This pipeline has only one stage:

```javascript
$bucket
```

`$bucket` groups documents into ranges.

---

### `groupBy`

```javascript
groupBy: "$age"
```

Means:

"Use the `age` field to decide which bucket a document belongs to."

The `$` means access a document field.

Example:

```javascript
"$age"
```

means:

```javascript
document.age
```

---

### `boundaries`

```javascript
boundaries: [0, 3, 9, 15]
```

Creates age groups:

```
0 - 2      Puppy
3 - 8      Adult
9 - 14     Senior
15+        Very senior
```

Important:

The last boundary is **exclusive**.

Meaning:

```
0 <= age < 3
3 <= age < 9
9 <= age < 15
```

Age `15` does not enter the last bucket, so it goes to:

```javascript
default: "very senior"
```

---

### `default`

```javascript
default: "very senior"
```

Handles documents that do not match any defined bucket.

Example:

```
Age: 20

Does not fit:

0-2
3-8
9-14

So:

very senior
```

---

### `output`

```javascript
output: {
  count: { $sum: 1 }
}
```

Defines what data each bucket should return.

`$sum: 1` means:

"For every document inside this bucket, increase the count by 1."

Example:

```
Dog 1 -> count = 1
Dog 2 -> count = 2
Dog 3 -> count = 3
```

---

# Adding `$match` to Filter Dogs Only

Currently the aggregation includes:

```
Dogs
Cats
Birds
Other pets
```

We only want dogs.

```javascript
db.pets.aggregate([
  {
    $match: {
      type: "dog",
    },
  },
  {
    $bucket: {
      groupBy: "$age",
      boundaries: [0, 3, 9, 15],
      default: "very senior",
      output: {
        count: { $sum: 1 },
      },
    },
  },
]);
```

### Result

Example:

```json
[
  {
    "_id": 0,
    "count": 5
  },
  {
    "_id": 3,
    "count": 10
  },
  {
    "_id": 9,
    "count": 4
  }
]
```

---

## Explanation

Now the pipeline has two stages:

```
Stage 1:
$match

Stage 2:
$bucket
```

The data flow:

```
All pets
   |
   v
$match
(remove non-dogs)
   |
   v
$bucket
(group dogs by age)
   |
   v
Result
```

---

## `$match`

```javascript
{
  $match: {
    type: "dog"
  }
}
```

Works like SQL:

```sql
WHERE type = 'dog'
```

It removes all documents that are not dogs before the next stage runs.

Benefits:

* Less data to process
* Faster aggregation
* Cleaner results

# Sorting Aggregation Results

Now we want the biggest group first.

```javascript
db.pets.aggregate([
  {
    $match: {
      type: "dog",
    },
  },
  {
    $bucket: {
      groupBy: "$age",
      boundaries: [0, 3, 9, 15],
      default: "very senior",
      output: {
        count: { $sum: 1 },
      },
    },
  },
  {
    $sort: {
      count: -1,
    },
  },
]);
```

### Result

Example:

```json
[
  {
    "_id": 3,
    "count": 10
  },
  {
    "_id": 0,
    "count": 5
  },
  {
    "_id": 9,
    "count": 4
  }
]
```

---

# Explanation

Now we have three stages:

```
Stage 1:
$match

Stage 2:
$bucket

Stage 3:
$sort
```

Pipeline:

```
All Pets
   |
   v
$match
Only dogs
   |
   v
$bucket
Group by age
   |
   v
$sort
Order by count
   |
   v
Final result
```

---

# `$sort`

```javascript
$sort: {
  count: -1
}
```

Sorts documents using the `count` field.

Values:

```
1  = ascending order
-1 = descending order
```

Example:

Ascending:

```
2
5
10
```

Descending:

```
10
5
2
```

Here:

```javascript
count: -1
```

means:

"Largest pet groups first."

# Common Aggregation Stages

Some frequently used stages:

| Stage      | Purpose                           |
| ---------- | --------------------------------- |
| `$match`   | Filter documents                  |
| `$group`   | Combine documents into groups     |
| `$bucket`  | Group values into ranges          |
| `$sort`    | Sort results                      |
| `$limit`   | Return only a number of documents |
| `$project` | Select or rename fields           |
| `$count`   | Count documents                   |
| `$lookup`  | Join another collection           |

---

# Aggregation Pipeline Mental Model

Think of aggregation like a chain:

```javascript
db.collection.aggregate([
  stage1,
  stage2,
  stage3
])
```

Each stage transforms the data.

Example:

```javascript
[
  filter,
  group,
  sort
]
```

means:

```
Take data
→ remove unwanted documents
→ group information
→ sort the result
```
