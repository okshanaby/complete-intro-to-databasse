# MongoDB Performance & Indexes — Simple Summary

When a MongoDB collection becomes large, queries can become slow if MongoDB has to examine every document.

**Indexes** help MongoDB find documents much faster by creating a data structure that allows quick lookups, similar to an index in a book.

Without an index, MongoDB performs a **collection scan (COLLSCAN)**, meaning it checks every document one by one.

## Key Points

* `explain()` shows how MongoDB executes a query.
* `COLLSCAN` means MongoDB scanned the entire collection.
* Indexes greatly improve query performance.
* Compound indexes optimize queries that frequently filter by multiple fields.
* Unique indexes prevent duplicate values.
* Text indexes enable full-text search.
* `getIndexes()` lists all indexes on a collection.

---

# Understanding `explain()`

Before optimizing a query, it's useful to see **how MongoDB executes it**.

Example query:

```javascript
db.pets.find({
  name: "Fido"
});
```

To view its execution plan:

```javascript
db.pets.find({
  name: "Fido"
}).explain("executionStats");
```

The `executionStats` output shows important information such as:

* Query execution strategy
* Number of documents examined
* Number of documents returned
* Execution time

---

# Collection Scan (COLLSCAN)

Without an index, MongoDB performs a **COLLSCAN**.

```text
Collection

Luna
Fido
Spot
Bella
Charlie
...

Searching for:
Fido

MongoDB checks:

Luna     ✗
Fido     ✓
Spot     ✗
Bella    ✗
Charlie  ✗
...
```

Every document is examined until MongoDB finishes searching.

This becomes slow as collections grow.

---

# Creating an Index

Create an index on the `name` field.

```javascript
db.pets.createIndex({
  name: 1
});
```

Now the same query:

```javascript
db.pets.find({
  name: "Fido"
}).explain("executionStats");
```

MongoDB can immediately locate matching documents instead of scanning the entire collection.

Benefits:

* Faster searches
* Fewer documents examined
* Better scalability

---

# Viewing Existing Indexes

To see every index on a collection:

```javascript
db.pets.getIndexes();
```

MongoDB returns a list of all indexes currently defined.

---

# Compound Indexes

Sometimes queries frequently search using **multiple fields together**.

Example:

```javascript
db.pets.find({
  type: "dog",
  breed: "Havanese"
});
```

Instead of creating two separate indexes, create one compound index.

```javascript
db.pets.createIndex({
  type: 1,
  breed: 1
});
```

This index is optimized for queries using both fields together.

Advantages:

* Faster multi-field searches.
* More efficient than multiple separate indexes for those queries.

---

# Unique Indexes

A unique index ensures no two documents have the same value.

Example:

```javascript
db.pets.createIndex(
  {
    index: 1
  },
  {
    unique: true
  }
);
```

Now MongoDB enforces uniqueness.

Trying to insert:

```javascript
db.pets.insertOne({
  name: "Doggo",
  index: 10
});
```

If another document already has:

```text
index: 10
```

MongoDB returns a **duplicate key error**.

Benefits:

* Prevents duplicate data.
* Automatically improves search performance on that field.

---

# Text Indexes

MongoDB supports **full-text search** using text indexes.

Create a text index:

```javascript
db.pets.createIndex({
  type: "text",
  breed: "text",
  name: "text"
});
```

A collection can have **only one text index**, so include every field you want searchable.

---

# Full-Text Search

Search for matching words using `$text`.

```javascript
db.pets.find({
  $text: {
    $search: "dog Havanese Luna"
  }
});
```

MongoDB searches across all fields included in the text index.

---

# Sorting by Relevance

Search results can be sorted by their relevance score.

```javascript
db.pets
  .find({
    $text: {
      $search: "dog Havanese Luna"
    }
  })
  .sort({
    score: {
      $meta: "textScore"
    }
  });
```

The documents most relevant to the search appear first.

---

# Showing the Search Score

To return the actual relevance score:

```javascript
db.pets
  .find(
    {
      $text: {
        $search: "dog Havanese Luna"
      }
    },
    {
      score: {
        $meta: "textScore"
      }
    }
  )
  .sort({
    score: {
      $meta: "textScore"
    }
  });
```

MongoDB adds a `score` field showing how closely each document matched the search.

---

# Excluding Search Terms

Text search also supports exclusions.

Example:

```javascript
db.pets
  .find({
    $text: {
      $search: "-cat Luna"
    }
  })
  .sort({
    score: {
      $meta: "textScore"
    }
  });
```

Meaning:

* Find documents containing **Luna**
* Exclude documents containing **cat**

# Vector Search

MongoDB also supports **vector search**, which is commonly used for AI and semantic search.

Instead of matching exact words, vector search finds documents with **similar meaning**.

The notes only mention this feature and do not explain it further.

# Types of Indexes

| Index Type         | Purpose                                           |
| ------------------ | ------------------------------------------------- |
| Single-field Index | Speeds up searches on one field                   |
| Compound Index     | Speeds up searches using multiple fields together |
| Unique Index       | Prevents duplicate values                         |
| Text Index         | Enables full-text searching                       |

---

# One-Sentence Summary

**Indexes improve MongoDB query performance by allowing the database to locate documents efficiently, while tools like `explain()` help analyze queries and specialized indexes such as compound, unique, and text indexes optimize different types of searches.**
