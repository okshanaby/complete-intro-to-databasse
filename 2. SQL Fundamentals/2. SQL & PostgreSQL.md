# SQL & PostgreSQL — Simple Summary

**SQL (Structured Query Language)** is the standard language used to communicate with **relational databases**. It allows you to create, read, update, and delete data (commonly called **CRUD** operations).

Although SQL is standardized, every database system (such as PostgreSQL, MySQL, or Oracle) has its own **dialect** with additional features and functions. This means SQL written for one database may need small changes to work on another.

This course focuses on **PostgreSQL**, which is one of the most popular and feature-rich open-source relational databases.

## Key Points

* SQL is the standard language for relational databases.
* SQL is used to:

  * Create data
  * Read data
  * Update data
  * Delete data
* Most relational databases support SQL.
* Different databases have slightly different SQL syntax and features.
* PostgreSQL is the database used throughout this course.

---

# What is SQL?

**SQL** stands for **Structured Query Language**.

It is a language used to send commands (queries) to relational databases.

Instead of sending JSON objects like MongoDB, you send SQL statements as plain text.

Example:

```sql
SELECT * FROM users;
```

The database reads the SQL statement and executes it.

## CRUD Operations

SQL is commonly used for CRUD operations:

| Operation | Description   | SQL Keyword |
| --------- | ------------- | ----------- |
| Create    | Add new data  | `INSERT`    |
| Read      | Retrieve data | `SELECT`    |
| Update    | Modify data   | `UPDATE`    |
| Delete    | Remove data   | `DELETE`    |

---

# Is SQL the Same Everywhere?

No.

SQL is a standard language, but each database has its own implementation.

Examples of SQL databases include:

* PostgreSQL
* MySQL
* MariaDB
* Oracle Database
* IBM DB2
* SQLite
* Microsoft SQL Server

Most basic SQL works across all of them, but advanced features may differ.

For example:

* PostgreSQL has PostgreSQL-specific functions.
* MySQL has MySQL-specific functions.
* Oracle has Oracle-specific features.

Because of these differences, SQL isn't always perfectly portable.

---

# SQL vs MongoDB Queries

MongoDB uses JSON-like query objects.

Example:

```javascript
db.users.find({ name: "Brian" });
```

SQL uses text-based queries.

```sql
SELECT * FROM users
WHERE name = 'Brian';
```

# What is PostgreSQL?

**PostgreSQL** (pronounced **"post-gress"**) is a free, open-source relational database.

It is known for:

* Reliability
* Excellent performance
* Advanced features
* Scalability
* Strong SQL support

Many large companies use PostgreSQL.

Examples include:

* Apple
* Microsoft
* Etsy

The instructor recommends PostgreSQL as the **default choice** for most applications because it is flexible and capable of handling many different workloads.

# SQLite

SQLite is different from traditional database servers.

Instead of running as a separate server process, SQLite is simply a **library** that stores data in a file.

Example:

```text
Application
      │
      ▼
 SQLite Library
      │
      ▼
database.db
```

Because there is no database server:

* No network communication
* Small size
* Very fast
* Easy to embed inside applications

SQLite is commonly found in:

* Smartphones
* Operating systems
* IoT devices
* Web browsers
* Game consoles
* Desktop applications

Some companies now even use SQLite as cloud databases.

Examples include:

* Turso
* Cloudflare

---

# Why is SQLite So Stable?

SQLite has one of the highest testing standards in software.

It reportedly has **10–100 lines of test code for every line of production code**, making it extremely reliable.

However, SQLite almost never breaks backward compatibility.

This means even old design decisions remain forever.

Example:

Foreign key constraints are **disabled by default** for each connection.

Developers must manually enable them.

Although this isn't ideal, SQLite keeps this behavior to avoid breaking older applications.

# Why PostgreSQL?

The instructor recommends PostgreSQL because it provides:

* Strong SQL support
* Excellent performance
* Advanced features
* Scalability
* Active open-source community
* Ability to support multiple workloads

PostgreSQL also supports features beyond traditional relational databases.

With extensions, it can handle:

* JSON documents
* Basic graph-like data
* Other specialized workloads

This reduces the need to use many different database systems.
