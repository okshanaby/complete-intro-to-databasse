# SELECT Queries in PostgreSQL — Simple Summary

SELECT queries are used to **read and retrieve data** from a database table. This is one of the most important parts of SQL because it lets you search, filter, and sort your data.

Before learning queries, the notes mention a **sample dataset reset script** that rebuilds your tables so you always start fresh. This is useful for practice.

---

# Basic SELECT (Get All Data)

```sql id="s1"
SELECT * FROM users;
```

---

## What this does

Returns **all columns and all rows** from the `users` table.

---

## Breakdown

### `SELECT`

* Tells PostgreSQL you want to read data.

### `*` (asterisk / wildcard)

* Means “everything”
* Selects all columns in the table

### `FROM users`

* Specifies which table to read from

---

## Important Note

This can return a lot of data (e.g., 1000+ rows), so it's often not efficient in real applications.

---

# LIMIT (Restrict Number of Rows)

```sql id="s2"
SELECT * FROM users LIMIT 10;
```

---

## What this does

Returns only the first **10 rows** from the table.

---

## Breakdown

### `LIMIT 10`

* Restricts the number of results
* Useful for testing and pagination

---

## Why it matters

Without LIMIT, a query could return thousands or millions of rows.

---

# Projection (Selecting Specific Columns)

```sql id="s3"
SELECT username, user_id FROM users LIMIT 15;
```

---

## What this does

Returns only the `username` and `user_id` columns for 15 users.

---

## Breakdown

### `username, user_id`

* These are the selected columns
* This is called **projection** (choosing only needed fields)

### `LIMIT 15`

* Restricts results to 15 rows

---

## Key Idea

Instead of:

```sql
SELECT * FROM users;
```

You should prefer:

```sql
SELECT username, email FROM users;
```

### Why?

* Faster queries
* Less memory usage
* Cleaner results

---

# WHERE (Filtering Data)

## Example 1: Find a specific user

```sql id="s4"
SELECT username, email, user_id
FROM users
WHERE user_id = 150;
```

---

### What it does

Finds the user whose `user_id` is exactly **150**.

---

### Breakdown

* `WHERE`

  * Filters rows based on a condition

* `user_id = 150`

  * Matches exact value

---

## Example 2: Find users with NULL last_login

```sql id="s5"
SELECT username, email, user_id
FROM users
WHERE last_login IS NULL
LIMIT 10;
```

---

### What it does

Finds users who have **never logged in**.

---

### Important Concept: NULL

* `NULL` means “no value”
* You cannot use `=` with NULL
* You must use:

```sql
IS NULL
```

---

# AND + Date Math

```sql id="s6"
SELECT username, email, user_id, created_on
FROM users
WHERE last_login IS NULL
  AND created_on < NOW() - interval '6 months'
LIMIT 10;
```

---

## What this does

Finds users who:

* Have never logged in
  AND
* Were created more than 6 months ago

---

## Breakdown

### `AND`

* Combines multiple conditions
* Both conditions must be true

---

### Condition 1

```sql
last_login IS NULL
```

User has never logged in.

---

### Condition 2 (Date math)

```sql
created_on < NOW() - interval '6 months'
```

#### Breakdown:

* `NOW()`

  * Current date and time

* `interval '6 months'`

  * Represents a time duration

* `NOW() - interval '6 months'`

  * Calculates the date 6 months ago

* `<`

  * Means “older than”

---

## Final Meaning

> “Give me users who haven’t logged in and are older than 6 months.”

---

# ORDER BY (Sorting Results)

## Oldest users first

```sql id="s7"
SELECT user_id, email, created_on
FROM users
ORDER BY created_on
LIMIT 10;
```

---

## What this does

Sorts users by creation date from **oldest to newest**, then returns 10 rows.

---

## Breakdown

### `ORDER BY created_on`

* Sorts results based on creation time
* Default is **ascending (ASC)**

So this means:

```text
oldest → newest
```

---

## Newest users first

```sql id="s8"
SELECT user_id, email, created_on
FROM users
ORDER BY created_on DESC
LIMIT 10;
```

---

## What this does

Returns the **10 most recently created users**.

---

## Breakdown

### `DESC`

* Means **descending order**
* Reverses sorting order

So this becomes:

```text
newest → oldest
```

---

### `ASC` (optional)

You can also explicitly write:

```sql
ORDER BY created_on ASC;
```

But:

* `ASC` is default
* So it’s usually omitted

---

# Key Concepts Summary

* `SELECT *` → returns all columns
* `LIMIT` → restricts number of rows returned
* Projection → selecting only needed columns
* `WHERE` → filters rows based on conditions
* `IS NULL` → checks for missing values
* `AND` → combines multiple conditions
* `NOW()` → current timestamp
* `interval 'x'` → represents time duration
* `ORDER BY` → sorts results
* `DESC` → descending order (newest first)

---

# One-Sentence Summary

**SELECT queries let you retrieve, filter, limit, and sort data from tables using clauses like WHERE, LIMIT, AND, and ORDER BY to precisely control what results you get.**
