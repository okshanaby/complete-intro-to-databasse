# Getting Started with PostgreSQL — Simple Summary

Before using PostgreSQL, you first need a PostgreSQL server running. In this tutorial, PostgreSQL is started inside a **Docker container**, so you don't have to install PostgreSQL directly on your computer.

Instead of using the standard PostgreSQL Docker image, the tutorial uses the **pgvector** image. It works exactly like PostgreSQL but already includes the **pgvector extension**, which is useful later for working with vector embeddings and AI-related features.

A password is required because PostgreSQL refuses to start without one by default.

After the container is running, you'll connect to PostgreSQL using the **psql** command-line tool.

## Start a PostgreSQL Container

```sh
docker run --name my-postgres \
  -e POSTGRES_PASSWORD=mysecretpassword \
  -p 5432:5432 \
  -d \
  --rm \
  pgvector/pgvector:pg18-trixie
```

### What this command does

This command downloads (if necessary) and starts a PostgreSQL Docker container.

Once started:

* The container is named **my-postgres**.
* PostgreSQL listens on port **5432**.
* A password is set for the default PostgreSQL user.
* The container runs in the background.
* The container is automatically removed after it is stopped.

### Command Breakdown

`docker run`

Starts a new Docker container from an image.

### Flags Used

`--name my-postgres`

* Gives the container the name **my-postgres**.
* Makes it easier to reference later.

Example:

```sh
docker stop my-postgres
```

---

`-e POSTGRES_PASSWORD=mysecretpassword`

* `-e` stands for **environment variable**.
* Sets the PostgreSQL password.
* PostgreSQL will not start unless a password is provided.

---

`-p 5432:5432`

* `-p` stands for **publish port**.
* Maps your computer's port **5432** to the container's port **5432**.

Format:

```text
host_port:container_port
```

Here:

* Left side = your computer
* Right side = Docker container

---

`-d`

* Runs the container in **detached mode**.
* This means it runs in the background instead of taking over your terminal.

---

`--rm`

* Automatically deletes the container after it is stopped.
* Useful for development since you don't leave unused containers behind.

---

`pgvector/pgvector:pg18-trixie`

This is the Docker image being used.

Parts:

```text
pgvector/pgvector
```

Repository name.

```text
pg18-trixie
```

Image tag.

It specifies:

* PostgreSQL 18
* Debian Trixie base image
* pgvector already installed

---

## Connect to PostgreSQL

```sh
docker exec -it -u postgres my-postgres psql
```

### What this command does

This opens the PostgreSQL interactive terminal (**psql**) inside the running Docker container.

You'll now be able to type SQL commands directly.

### Command Breakdown

`docker exec`

Runs a command inside an already-running container.

### Flags Used

`-i`

* Keeps standard input open.
* Lets you type commands interactively.

---

`-t`

* Allocates a terminal (TTY).
* Makes the session behave like a normal terminal.

You'll often see these together:

```sh
-it
```

which simply means:

* Interactive terminal.

---

`-u postgres`

* `-u` means **user**.
* Runs the command as the **postgres** user inside the container.

---

`my-postgres`

The name of the running Docker container.

---

`psql`

Launches PostgreSQL's interactive SQL shell.

Once connected, your prompt will look something like:

```text
postgres=#
```

---

# Databases and Tables

The word **database** can have two meanings:

1. A database system like PostgreSQL or MySQL.
2. A single database **inside** PostgreSQL.

PostgreSQL can contain **multiple databases**.

For example:

```text
PostgreSQL
│
├── analytics
├── operational
├── message_boards
└── inventory
```

Each database is a separate collection of related tables.

## Spreadsheet Analogy

Think of it like Microsoft Excel.

```text
PostgreSQL
    ↓
Workbook

Database
    ↓
One workbook/file

Table
    ↓
Worksheet (tab)

Row (Record)
    ↓
One entry

Column
    ↓
One field of information
```

For example:

Database:

```text
message_boards
```

Table:

```text
users
```

Rows:

```text
Alice
Bob
Charlie
```

Columns:

```text
id
name
email
```

---

## Tables

A table stores related information.

Example:

```text
users

+----+---------+----------------------+
| id | name    | email                |
+----+---------+----------------------+
| 1  | Alice   | alice@email.com      |
| 2  | Bob     | bob@email.com        |
+----+---------+----------------------+
```

Each row is called a **record**.

Each column stores one specific type of information.

---

## Schema

Every table has a **schema**.

A schema defines:

* Column names
* Data types
* Structure of the table

Example:

```text
users

id      INTEGER
name    TEXT
email   TEXT
```

Unlike spreadsheets, you cannot freely add columns whenever you want without modifying the table.

Changing a schema on very large tables can take a long time, so planning your database structure ahead of time is important.

---

# Writing SQL

SQL keywords are usually written in **UPPERCASE**.

Example:

```sql
CREATE DATABASE message_boards;
```

This isn't required, but it's a common convention because it makes SQL easier to read.

---

# SQL Statements End with a Semicolon

Unlike JavaScript, SQL statements normally end with a semicolon.

Example:

```sql
CREATE DATABASE message_boards;
```

Without the semicolon:

```sql
CREATE DATABASE message_boards
```

PostgreSQL assumes you're still typing the statement and waits for more input.

---

# Create Your First Database

```sql
CREATE DATABASE message_boards;
```

### What this command does

Creates a new database named **message_boards**.

### Command Breakdown

`CREATE DATABASE`

SQL command used to create a new database.

`message_boards`

The name of the database being created.

After running it successfully, PostgreSQL displays:

```text
CREATE DATABASE
```

This confirms that the database was created.

---

# Connect to the New Database

```sql
\c message_boards;
```

Alternative command:

```sql
\connect message_boards
```

### What this command does

Switches your current connection to the **message_boards** database.

### Command Breakdown

`\c`

Short for **connect**.

This is a **psql command**, not standard SQL.

`message_boards`

The database you want to connect to.

After connecting successfully, all future SQL commands will run inside this database.

---

# View All Databases

```sql
\l
```

### What this command does

Lists every database available in the PostgreSQL server.

### Command Breakdown

`\l`

Short for **list databases**.

This is a **psql command**, not SQL.

---

# View All Tables

```sql
\d
```

### What this command does

Displays all tables in the currently connected database.

Since the database is new, it will probably be empty.

### Command Breakdown

`\d`

Short for **describe**.

Shows database objects such as tables.

---

# Show All psql Commands

```sql
\?
```

### What this command does

Displays all available **psql** commands.

These commands are used to control the PostgreSQL terminal itself rather than execute SQL.

### Command Breakdown

`\?`

Shows the help menu for psql commands.

---

# Show SQL Help

```sql
\h
```

### What this command does

Displays help for SQL commands supported by PostgreSQL.

### Command Breakdown

`\h`

Shows SQL syntax help.

For example, you can look up how to use commands like:

* `CREATE TABLE`
* `INSERT`
* `SELECT`
* `UPDATE`

---

# Run a Shell Command

```sql
\! ls && echo "hi from shell!"
```

### What this command does

Runs a normal operating system shell command without leaving the PostgreSQL terminal.

In this example:

1. Lists the current directory.
2. Prints a message.

Example output:

```text
file1
file2
notes.txt
hi from shell!
```

### Command Breakdown

`\!`

Tells **psql** to execute a shell command.

`ls`

Lists files in the current directory (Linux/macOS).

`&&`

Runs the second command only if the first command succeeds.

`echo`

Prints text to the terminal.

---

# SQL Comments

SQL uses `--` for single-line comments.

Example:

```sql
-- This creates a new database
CREATE DATABASE message_boards;
```

Everything after `--` on the same line is ignored by PostgreSQL.

---

# Key Points

* PostgreSQL is being run inside a Docker container.
* The tutorial uses the **pgvector** image because it already includes the pgvector extension.
* PostgreSQL requires a password before it will start.
* `psql` is PostgreSQL's interactive command-line interface.
* PostgreSQL can contain multiple databases.
* A database is a collection of related tables.
* Tables store rows (records) and columns (fields).
* Every table has a predefined schema.
* SQL statements normally end with a semicolon (`;`).
* Commands beginning with `\` are **psql commands**, not SQL commands.
* `--` is used for single-line comments in SQL.

# One-Sentence Summary

**Start PostgreSQL in a Docker container, connect to it with `psql`, create and switch to a new database using SQL, and use `psql` commands to explore databases, tables, help menus, and even run shell commands while learning the basic structure of PostgreSQL.**
