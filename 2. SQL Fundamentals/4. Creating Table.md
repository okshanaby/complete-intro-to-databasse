# Creating Tables in PostgreSQL — Simple Summary

After connecting to your database, the next step is to create **tables**, which store your actual data in rows and columns.

A table is like a spreadsheet sheet:

* Columns = fields (like username, email)
* Rows = records (like one user)

---

# Create a Table

```sql
CREATE TABLE users (
  user_id INTEGER PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  username VARCHAR (25) UNIQUE NOT NULL,
  email VARCHAR (50) UNIQUE NOT NULL,
  full_name VARCHAR (100) NOT NULL,
  last_login TIMESTAMP,
  created_on TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

---

## What this command does

This creates a table named **users** with several columns and rules (constraints) that define what data is allowed.

---

## Command Breakdown

### `CREATE TABLE users`

Creates a new table called `users` inside the current database.

---

## Columns Explained

### 1. `user_id INTEGER PRIMARY KEY GENERATED ALWAYS AS IDENTITY`

```text
user_id INTEGER PRIMARY KEY GENERATED ALWAYS AS IDENTITY
```

### What it does

* Creates a numeric ID for each user.
* Automatically increases for each new row (1, 2, 3, ...).
* Ensures each user has a unique identifier.

### Breakdown

* `INTEGER`

  * The column stores whole numbers.

* `PRIMARY KEY`

  * Uniquely identifies each row in the table.
  * Cannot be NULL.
  * Automatically indexed for fast lookup.

* `GENERATED ALWAYS AS IDENTITY`

  * Automatically generates a new number for each row.
  * Modern replacement for the older `SERIAL` type.
  * Recommended for new PostgreSQL projects.

---

## Important Note

PostgreSQL does NOT automatically create a primary key.

You must define it manually if you want one.

---

### 2. `username VARCHAR(25) UNIQUE NOT NULL`

```text
username VARCHAR(25) UNIQUE NOT NULL
```

### What it does

Stores the username.

### Breakdown

* `VARCHAR(25)`

  * Variable-length string.
  * Maximum of 25 characters.

* `UNIQUE`

  * No two users can have the same username.

* `NOT NULL`

  * Value must be provided.
  * Cannot be empty or missing.

---

### 3. `email VARCHAR(50) UNIQUE NOT NULL`

```text
email VARCHAR(50) UNIQUE NOT NULL
```

Same rules as username, but:

* Maximum length: 50 characters
* Must also be unique

---

### 4. `full_name VARCHAR(100) NOT NULL`

```text
full_name VARCHAR(100) NOT NULL
```

### What it does

Stores the user’s full name.

### Breakdown

* `VARCHAR(100)` → allows longer names
* `NOT NULL` → must always be provided
* Not unique → multiple users can share the same name

Example:

* Sally Rogers (can appear many times)

---

### 5. `last_login TIMESTAMP`

```text
last_login TIMESTAMP
```

### What it does

Stores the last time a user logged in.

### Important

* No `NOT NULL`
* Can be `NULL` initially

Meaning:

* New users may not have logged in yet

---

### 6. `created_on TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP`

```text
created_on TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
```

### What it does

Stores when the user was created.

### Breakdown

* `TIMESTAMP`

  * Stores date + time

* `NOT NULL`

  * Must always have a value

* `DEFAULT CURRENT_TIMESTAMP`

  * If no value is provided, PostgreSQL automatically fills it with the current time.

---

## `CURRENT_TIMESTAMP` vs `NOW()`

Both return the current time:

```text
CURRENT_TIMESTAMP (SQL standard)
NOW() (PostgreSQL-specific)
```

They behave the same in practice, but:

* `CURRENT_TIMESTAMP` → standard SQL (preferred for portability)
* `NOW()` → PostgreSQL convenience function

# Insert a Record (Add a Row)

```sql
INSERT INTO users (username, email, full_name)
VALUES ('btholt', 'lol@example.com', 'Brian Holt');
```

---

## What this command does

Adds a new user (row) into the `users` table.

---

## Command Breakdown

### `INSERT INTO users`

Tells PostgreSQL:

* You are adding data into the `users` table.

---

### `(username, email, full_name)`

Defines **which columns you are inserting into**.

Important:

* Order matters.
* Must match the VALUES order.

---

### `VALUES (...)`

```text
('btholt', 'lol@example.com', 'Brian Holt')
```

These are the actual values being inserted.

---

## Why some fields are missing

You did NOT include:

* `user_id`
* `last_login`
* `created_on`

### Why?

* `user_id` → auto-generated (IDENTITY)
* `last_login` → optional (can be NULL)
* `created_on` → automatically set using DEFAULT CURRENT_TIMESTAMP

---

## Result

```text
INSERT 0 1
```

### Meaning

* `1` → one row was successfully inserted
* `0` → internal PostgreSQL OID (not important for this course)

---

# View All Records

```sql
SELECT * FROM users;
```

---

## What this command does

Fetches and displays **all rows and all columns** from the `users` table.

---

## Command Breakdown

### `SELECT`

Used to retrieve data from a table.

---

### `*`

Means:

* “Select everything”
* All columns

---

### `FROM users`

Specifies which table to read from.

---

## Result

You will see the inserted user:

```text
user_id | username | email           | full_name    | last_login | created_on
--------+----------+-----------------+--------------+------------+------------
1       | btholt   | lol@example.com | Brian Holt   | NULL       | 2026-07-06
```

---

# Key Concepts Summary

* `CREATE TABLE` defines structure for storing data.
* Tables are made of:

  * Columns (fields)
  * Rows (records)
* `PRIMARY KEY` uniquely identifies each row.
* `GENERATED ALWAYS AS IDENTITY` auto-increments IDs.
* `VARCHAR(n)` limits string length.
* `UNIQUE` prevents duplicates.
* `NOT NULL` forces a value to exist.
* `DEFAULT` sets automatic values when none are provided.
* `INSERT INTO` adds new rows.
* `SELECT *` retrieves all data from a table.
