# Subqueries & GROUP BY in PostgreSQL — Simple Summary

This section introduces two powerful SQL ideas:

* **Subqueries** → a query inside another query
* **GROUP BY** → summarizing and grouping data

These help you write more advanced and efficient queries.

---

# 1. Subqueries (Query inside a query)

A **subquery** is a SQL query executed *inside another query*.

* Inner query runs first
* Its result is used by the outer query

---

## Example Problem

> Find all comments made by “Maynord Simonich”

You could do it in two steps:

1. Get user_id from users table
2. Use that user_id in comments table

Or do it in one query using a subquery.

---

## Subquery Example

```sql id="sq1"
SELECT comment_id, user_id, LEFT(comment, 20)
FROM comments
WHERE user_id = (
  SELECT user_id
  FROM users
  WHERE full_name = 'Maynord Simonich'
);
```

---

## How it works

### Inner query (runs first)

```sql id="sq2"
SELECT user_id
FROM users
WHERE full_name = 'Maynord Simonich'
```

* Finds the user ID of that person

---

### Outer query (runs second)

```sql id="sq3"
SELECT comment_id, user_id, ...
FROM comments
WHERE user_id = (result from inner query)
```

* Uses the returned user_id to filter comments

---

## Important Rule

### `= (subquery)` requires ONE value

* Must return exactly **one row**
* If it returns:

  * 0 rows → error
  * multiple rows → error

---

## If multiple values are possible

Use `IN` instead:

```sql id="sq4"
WHERE user_id IN (SELECT user_id FROM users)
```

---

## Key Idea

> Subqueries let you “nest logic” when one query depends on another.

---

# 2. GROUP BY (Grouping + Aggregation)

`GROUP BY` is used to:

* Group rows with the same value
* Perform calculations on each group

Common with:

* `COUNT`
* `SUM`
* `AVG`

---

## Example: Most active message boards

```sql id="gb1"
SELECT
  boards.board_name,
  COUNT(*) AS comment_count
FROM comments
INNER JOIN boards
ON boards.board_id = comments.board_id
GROUP BY boards.board_name
ORDER BY comment_count DESC
LIMIT 10;
```

---

## What this query does

Finds the **top 10 boards with the most comments**.

---

## Breakdown

### JOIN step

```text id="gb2"
comments + boards
```

Combines comments with board names.

---

### GROUP BY

```sql id="gb3"
GROUP BY boards.board_name
```

* Groups all rows by board name
* Each group = one board

---

### COUNT(*)

```sql id="gb4"
COUNT(*)
```

* Counts number of rows in each group
* = number of comments per board

---

### ORDER BY

```sql id="gb5"
ORDER BY comment_count DESC
```

* Sorts from most comments → least comments

---

### LIMIT 10

* Returns only top 10 boards

---

# Problem: Boards with ZERO comments

Now try finding least active boards:

```sql id="gb6"
SELECT
  boards.board_name,
  COUNT(*) AS comment_count
FROM comments
INNER JOIN boards
ON boards.board_id = comments.board_id
GROUP BY boards.board_name
ORDER BY comment_count ASC
LIMIT 10;
```

---

## Problem with this query

Because of:

```sql id="gb7"
INNER JOIN
```

👉 Boards with **no comments are excluded**

So empty boards NEVER appear.

---

# Fix: Use RIGHT JOIN

```sql id="gb8"
SELECT
  boards.board_name,
  COUNT(comment_id) AS comment_count
FROM comments
RIGHT JOIN boards
ON boards.board_id = comments.board_id
GROUP BY boards.board_name
ORDER BY comment_count;
```

---

## What changed?

### 1. RIGHT JOIN

```text id="gb9"
RIGHT JOIN boards
```

* Keeps ALL boards
* Even if they have no comments

---

## JOIN direction idea

* LEFT table → comments
* RIGHT table → boards

So RIGHT JOIN ensures:

> “Keep all boards no matter what”

---

### 2. COUNT(comment_id) instead of COUNT(*)

```sql id="gb10"
COUNT(comment_id)
```

### Why?

* `COUNT(*)` counts rows even if comment is NULL
* Empty boards would incorrectly show as 1

---

## Correct behavior

| Board        | comment_id values | COUNT(comment_id) |
| ------------ | ----------------- | ----------------- |
| Active board | 5 comments        | 5                 |
| Empty board  | NULL              | 0                 |

---

# Key Concepts Summary

* Subquery = query inside another query
* Subqueries run first and feed results to outer query
* `=` subquery must return ONE value
* Use `IN` for multiple values
* `GROUP BY` groups rows with same value
* Aggregations (COUNT, AVG, SUM) work per group
* `INNER JOIN` excludes unmatched rows
* `RIGHT JOIN` includes all rows from right table
* `COUNT(*)` counts rows, even NULLs
* `COUNT(column)` ignores NULL values

---

# One-Sentence Summary

**Subqueries let you nest queries to reuse results, while GROUP BY lets you summarize data into groups using aggregations like COUNT, making it possible to analyze patterns like top or empty message boards.**
