# COUNT, UPDATE, DELETE in PostgreSQL — Simple Summary

These three SQL commands let you **analyze, modify, and remove data** from a database:

* `COUNT` → measure data
* `UPDATE` → change data
* `DELETE` → remove data

---

# COUNT (How many rows exist?)

## Count all rows

```sql id="c1"
SELECT COUNT(*) FROM users;
```

---

## What this does

Returns the **total number of rows** in the `users` table.

---

## Breakdown

### `COUNT(*)`

* Counts **all rows**
* Includes rows even if some columns are NULL

### `*`

* Means “all rows”

---

## Example meaning

> “How many users exist in the table?”

---

# COUNT specific column (ignores NULLs)

```sql id="c2"
SELECT COUNT(last_login) FROM users;
```

---

## What this does

Counts only users who have a value in `last_login`.

---

## Important concept: NULL behavior

* `COUNT(column)` ignores NULL values
* So users who never logged in are NOT counted

---

## Example meaning

> “How many users have ever logged in?”

---

# UPDATE (Modify existing data)

## Update one field

```sql id="u1"
UPDATE users
SET last_login = NOW()
WHERE user_id = 1
RETURNING *;
```

---

## What this does

Updates the `last_login` field for user with `user_id = 1`, then returns the updated row.

---

## Breakdown

### `UPDATE users`

* Specifies the table to modify

---

### `SET last_login = NOW()`

* `SET` defines what to change
* `NOW()` sets current timestamp

---

### `WHERE user_id = 1`

* Targets only one specific row
* VERY IMPORTANT (prevents updating all users)

---

### `RETURNING *`

* Returns the updated row after modification
* Useful for confirming changes immediately

---

## What happens if WHERE is missing?

```sql id="u2"
UPDATE users SET last_login = NOW();
```

This would update ALL users.

---

# UPDATE multiple fields

```sql id="u3"
UPDATE users
SET full_name = 'Brian Holt',
    email = 'lol@example.com'
WHERE user_id = 2
RETURNING *;
```

---

## What this does

Updates:

* `full_name`
* `email`

for a single user.

---

## Breakdown

### Multiple fields in SET

```text id="u3b"
full_name = 'Brian Holt',
email = 'lol@example.com'
```

* Separate multiple updates using commas

---

## Important rule

* Use **single quotes `' '` for strings**
* Double quotes `" "` are NOT for values (they cause errors in PostgreSQL)

---

## Example meaning

> “Update user #2 with new name and email, then show me the updated record.”

---

# DELETE (Remove data)

## Delete a row

```sql id="d1"
DELETE FROM users
WHERE user_id = 1000;
```

---

## What this does

Removes the user with `user_id = 1000` from the table.

---

## Breakdown

### `DELETE FROM users`

* Specifies the table to delete from

---

### `WHERE user_id = 1000`

* Ensures only one specific user is deleted

---

## Important warning

If you omit `WHERE`:

```sql id="d2"
DELETE FROM users;
```

This will delete **ALL rows in the table**.

---

# Key Concepts Summary

* `COUNT(*)` → counts all rows
* `COUNT(column)` → counts only non-NULL values
* `UPDATE` → modifies existing data
* `SET` → defines new values
* `WHERE` → selects which rows to change or delete
* `RETURNING *` → shows updated/deleted rows
* `DELETE` → removes rows from a table
* Always use `WHERE` carefully to avoid affecting all rows

# NOTE: 

An **aggregation** is an operation that takes multiple rows and condenses them down into fewer rows or a single answer. For example, using `COUNT(*)` to get a total count instead of returning all individual rows is an aggregation.

In **SQL**, string literals should always use single quotes. For example: `UPDATE users SET full_name = 'Brian' WHERE user_id = 1`;

---


# One-Sentence Summary

**COUNT lets you measure data, UPDATE lets you modify existing records, and DELETE removes records—each relying heavily on WHERE to safely target specific rows.**
