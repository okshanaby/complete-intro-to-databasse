# Getting Started with Neo4j — Simple Summary

In these notes, you'll learn how to:

* Run a Neo4j database using Docker.
* Connect to the database using the **Cypher Shell**.
* Use **Cypher**, Neo4j's query language.
* Create nodes (People and Movies).
* Create relationships between nodes.
* Query graph data.
* Create uniqueness constraints.

Unlike SQL, where you work with **tables**, Neo4j stores **nodes** and **relationships**, making it much easier to model connected data.

---

# Step 1 — Start Neo4j with Docker

Run the following command:

```bash
docker run -dit --rm --name=my-neo4j \
-p 7474:7474 \
-p 7687:7687 \
--env=NEO4J_AUTH=none \
neo4j:2026-community-trixie
```

### Result

A new Neo4j database container starts in the background.

You can now access:

* Neo4j Browser (Web UI): [http://localhost:7474](http://localhost:7474)
* Bolt (database connection): `localhost:7687`

---

## What this command does

This command downloads (if needed) and starts a Neo4j Docker container.

### Command breakdown

```bash
docker run
```

Starts a new Docker container.

Flags:

* `-d`

  * Runs the container in **detached mode** (background).

* `-i`

  * Keeps STDIN open.

* `-t`

  * Allocates a terminal.

Together:

```bash
-dit
```

means:

* Run in background
* Keep terminal available if needed

---

```bash
--rm
```

Automatically removes the container after it stops.

Without this flag, stopped containers remain on your system.

---

```bash
--name=my-neo4j
```

Names the container:

```
my-neo4j
```

This makes it easier to reference later.

---

```bash
-p 7474:7474
```

Maps:

```
Host Port 7474
      ↓
Container Port 7474
```

Port **7474** is the Neo4j Browser (web interface).

---

```bash
-p 7687:7687
```

Maps the Bolt database protocol.

Applications connect to Neo4j through this port.

---

```bash
--env=NEO4J_AUTH=none
```

Sets an environment variable.

Normally Neo4j asks for a username/password.

This disables authentication for learning purposes.

---

```bash
neo4j:2026-community-trixie
```

The Docker image being used.

* `neo4j`

  * Image name

* `2026-community-trixie`

  * Version/tag

# Step 2 — Connect to Neo4j

Run:

```bash
docker exec -it my-neo4j cypher-shell
```

### Result

You'll enter the interactive Neo4j command-line shell:

```text
neo4j>
```

You can now execute Cypher queries.

---

## What this command does

```bash
docker exec
```

Runs a command inside an already-running container.

Flags:

* `-i`

  * Interactive mode.

* `-t`

  * Allocates a terminal.

---

```bash
my-neo4j
```

The name of the running container.

---

```bash
cypher-shell
```

Starts Neo4j's command-line client.

Think of it like:

* PostgreSQL → `psql`
* MongoDB → `mongosh`
* Neo4j → `cypher-shell`

# Cypher

Cypher is Neo4j's query language.

Just like:

* PostgreSQL → SQL
* Neo4j → Cypher

Cypher is designed to describe graphs naturally.

---

# Creating Your First Node

```cypher
CREATE (:Person {name:'Michael Cera', born:1988});
```

### Result

Creates one **Person** node:

```text
(Person)
name: Michael Cera
born: 1988
```

---

## What this query does

Creates a new node with:

* Label: `Person`
* Name
* Birth year

---

### Breaking it down

```cypher
CREATE
```

Creates new data.

---

```cypher
:Person
```

The `:` indicates a **label**.

Without the colon, Neo4j treats it as a variable instead of a label.

---

```cypher
{
    name:'Michael Cera',
    born:1988
}
```

These are the node's **properties**.

Similar to columns in SQL.

---

# Finding a Node

```cypher
MATCH (p {name: "Michael Cera"})
RETURN p;
```

### Result

Returns the previously created Person node.

---

## What this query does

Searches for a node whose `name` property is **Michael Cera** and returns it.

---

### Breaking it down

```cypher
MATCH
```

Searches the graph.

Comparable to SQL's `SELECT`.

---

```cypher
(p)
```

Creates a variable named `p`.

It could be named anything:

```cypher
(actor)
(movie)
(x)
(person)
```

---

```cypher
{name:"Michael Cera"}
```

Filters by property.

---

```cypher
RETURN p
```

Displays the result.

Without `RETURN`, nothing is shown.

---

# Creating a Movie

```cypher
CREATE (m:Movie {
    title: 'Scott Pilgrim vs the World',
    released: 2010,
    tagline: 'An epic of epic epicness.'
})
RETURN m;
```

### Result

Creates and immediately returns a Movie node.

---

## What this query does

Creates a node labeled **Movie** with:

* title
* release year
* tagline

and returns it.

---

### Breaking it down

```cypher
m:Movie
```

* `m`

  * Variable

* `Movie`

  * Label

---

```cypher
RETURN m
```

Displays the newly created node.

---

# Creating a Relationship

```cypher
MATCH (Michael:Person),(ScottVsWorld:Movie)
WHERE Michael.name = "Michael Cera"
AND ScottVsWorld.title = "Scott Pilgrim vs the World"
CREATE (Michael)-[relationship:ACTED_IN {roles:["Scott Pilgrim"]}]->(ScottVsWorld)
RETURN relationship;
```

### Result

Creates this graph:

```text
Michael Cera
      |
 ACTED_IN
      |
      V
Scott Pilgrim vs the World
```

The relationship also stores:

```text
roles = ["Scott Pilgrim"]
```

---

## What this query does

1. Finds Michael Cera.
2. Finds the movie.
3. Creates an `ACTED_IN` relationship.
4. Stores the role played.
5. Returns the relationship.

---

### Breaking it down

```cypher
MATCH
```

Finds existing nodes.

---

```cypher
(Michael:Person)
```

Variable:

```
Michael
```

Label:

```
Person
```

---

```cypher
(ScottVsWorld:Movie)
```

Finds a Movie.

---

```cypher
WHERE
```

Filters the matched nodes.

---

```cypher
CREATE
```

Creates a relationship.

---

```cypher
(Michael)
```

Starting node.

---

```cypher
-[relationship:ACTED_IN {roles:["Scott Pilgrim"]}]->
```

Creates:

* Variable: `relationship`
* Type: `ACTED_IN`
* Property:

```text
roles = ["Scott Pilgrim"]
```

---

```cypher
->
```

Shows the direction:

```
Person
   ↓
Movie
```

---

```cypher
RETURN relationship
```

Returns the new relationship.

# Adding More Actors and the Director

```cypher
MATCH (ScottVsWorld:Movie)
WHERE ScottVsWorld.title = "Scott Pilgrim vs the World"

CREATE (Anna:Person {name:'Anna Kendrick', born:1985})
CREATE (Brie:Person {name:'Brie Larson', born:1989})
CREATE (Aubrey:Person {name:'Aubrey Plaza', born:1984})
CREATE (Mary:Person {name:'Mary Elizabeth Winstead', born:1984})
CREATE (Kieran:Person {name:'Kieran Culkin', born:1982})
CREATE (Chris:Person {name:'Chris Evans', born:1981})
CREATE (Edgar:Person {name:'Edgar Wright', born:1974})

CREATE
(Anna)-[:ACTED_IN {roles:['Stacey Pilgrim']}]->(ScottVsWorld),
(Brie)-[:ACTED_IN {roles:['Envy Adams']}]->(ScottVsWorld),
(Aubrey)-[:ACTED_IN {roles:['Julie Powers']}]->(ScottVsWorld),
(Mary)-[:ACTED_IN {roles:['Ramona Flowers']}]->(ScottVsWorld),
(Kieran)-[:ACTED_IN {roles:['Wallace Wells']}]->(ScottVsWorld),
(Chris)-[:ACTED_IN {roles:['Lucas Lee']}]->(ScottVsWorld),
(Edgar)-[:DIRECTED]->(ScottVsWorld);
```

### Result

Creates:

* 7 new Person nodes.
* 6 `ACTED_IN` relationships.
* 1 `DIRECTED` relationship.

---

## What this query does

It first finds the movie, then creates several people and immediately connects each one to the movie with the appropriate relationship. Cypher allows multiple `CREATE` statements in a single query, making it easy to build a larger graph at once.

---

# Find Movies an Actor Appeared In

```cypher
MATCH (p:Person)-[:ACTED_IN]->(m:Movie)
WHERE p.name = "Aubrey Plaza"
RETURN m.title;
```

### Result

```text
Scott Pilgrim vs the World
```

---

## What this query does

Starts from the **Person** node, follows the `ACTED_IN` relationship to a **Movie** node, filters for Aubrey Plaza, and returns the movie title.

---

### Relationship Pattern

```text
(Person) --ACTED_IN--> (Movie)
```

This is one of Cypher's strengths—queries often read like a diagram of the graph.

---

# Find Everyone Who Acted with Aubrey Plaza

```cypher
MATCH (p:Person)-[:ACTED_IN]->(Movie)<-[:ACTED_IN]-(q:Person)
WHERE p.name = "Aubrey Plaza"
AND q.name <> "Aubrey Plaza"
RETURN q.name;
```

### Result

Returns the names of every other actor who acted in the same movie as Aubrey Plaza.

---

## What this query does

The query:

1. Finds Aubrey Plaza.
2. Follows the `ACTED_IN` relationship to the movie.
3. Finds other people connected to the same movie through `ACTED_IN`.
4. Excludes Aubrey herself using `<>` (not equal).

---

### Relationship Pattern

```text
(Person)
     |
ACTED_IN
     |
     V
(Movie)
     ^
     |
ACTED_IN
     |
(Person)
```

This is a common graph traversal: finding nodes connected through another node.

---

# Find Younger Co-stars

```cypher
MATCH (p:Person)-[:ACTED_IN]->(Movie)<-[:ACTED_IN]-(q:Person)
WHERE p.name = "Aubrey Plaza"
AND q.born > p.born
RETURN q.name;
```

### Result

Returns actors who appeared in the same movie and were born after Aubrey Plaza (i.e., younger than her).

---

## What this query does

This is similar to the previous query, but instead of excluding Aubrey by name, it compares the `born` property to find only younger co-stars.

---

# Creating Constraints

```cypher
CREATE CONSTRAINT FOR (a:Person)
REQUIRE a.name IS UNIQUE;
```

### Result

Creates a uniqueness constraint on the `Person` label so that no two `Person` nodes can have the same `name`.

---

## What this query does

A **constraint** enforces a rule on the data. Here, it ensures every `Person` node has a unique `name`.

> In a real application, using a person's name as a unique identifier is usually not ideal because different people can share the same name.

---

```cypher
CREATE CONSTRAINT FOR (a:Movie)
REQUIRE a.title IS UNIQUE;
```

### Result

Creates a uniqueness constraint on the `Movie` label so that no two `Movie` nodes can have the same `title`.

---

## What this query does

This prevents duplicate movie titles from being inserted. As with people, this is only an example—different movies can sometimes share the same title.

---

# Neo4j Constraint Syntax

Neo4j 5 introduced a new constraint syntax.

Older versions used:

* `ON`
* `ASSERT`

Neo4j 5 and later use:

* `FOR`
* `REQUIRE`

So the modern syntax is:

```cypher
CREATE CONSTRAINT FOR (n:Label)
REQUIRE n.property IS UNIQUE;
```

# Key Points

* Use Docker to quickly start a Neo4j database.
* Connect to Neo4j with `cypher-shell`.
* Cypher is Neo4j's graph query language.
* `CREATE` creates nodes and relationships.
* `MATCH` searches existing nodes and relationships.
* `RETURN` displays query results.
* Labels (such as `Person` and `Movie`) define the type of a node.
* Relationships (such as `ACTED_IN` and `DIRECTED`) connect nodes and can store their own properties.
* Relationship direction is represented using `->`, but queries can ignore direction when appropriate.
* `WHERE` filters matched data.
* `<>` is the "not equal" operator in Cypher.
* Constraints help maintain data integrity by enforcing rules such as uniqueness.

# One-Sentence Summary

**Neo4j uses the Cypher query language to create, connect, and query nodes through relationships, making it easy to model and traverse highly connected data while maintaining data integrity with constraints.**
