# Complex Neo4j Queries — Simple Summary

These notes introduce more advanced **Cypher** queries. Instead of simply creating and finding nodes, you'll learn how to:

* Count nodes and relationships.
* Group and aggregate data.
* Find actors who worked together.
* Sort results.
* Display graph visualizations.
* Find the shortest path between two people.
* Expand paths into readable results.
* Search through multiple relationship levels (degrees of separation).

These are some of the features that make graph databases very powerful.

---

# Count Nodes by Label

```cypher
MATCH (n)
RETURN DISTINCT labels(n), count(*);
```

### Result

Returns something similar to:

```text
labels(n)     count
-----------   -----
["Person"]      8
["Movie"]       1
```

---

## What this query does

It finds **every node** in the graph, groups them by their labels, and counts how many nodes belong to each label.

---

### Breaking it down

```cypher
MATCH (n)
```

Matches **all nodes**.

Notice there is **no label** after `n`.

This means:

```text
Any node in the graph
```

---

```cypher
labels(n)
```

Built-in function.

Returns the labels assigned to a node.

Example:

```text
Person
Movie
```

---

```cypher
count(*)
```

Counts the number of matching rows.

Similar to SQL:

```sql
COUNT(*)
```

---

```cypher
DISTINCT
```

Removes duplicate label groups before displaying results.

---

# Count Relationship Types

```cypher
MATCH (n)-[r]->()
RETURN type(r), count(*);
```

### Result

Example:

```text
type(r)     count
----------  -----
ACTED_IN       7
DIRECTED       1
```

---

## What this query does

Counts how many relationships exist for each relationship type.

---

### Breaking it down

```cypher
(n)-[r]->()
```

Means:

* Any starting node
* Any relationship
* Any ending node

---

```cypher
type(r)
```

Returns the relationship type.

Example:

```text
ACTED_IN
DIRECTED
```

---

```cypher
count(*)
```

Counts each relationship type.

# Find Keanu Reeves' Co-stars

```cypher
MATCH (Keanu:Person)-[:ACTED_IN]->(m:Movie)<-[:ACTED_IN]-(Costar:Person)
WHERE Keanu.name = "Keanu Reeves"
RETURN DISTINCT Costar.name, count(*)
ORDER BY count(*) DESC, Costar.name;
```

### Result

Example:

```text
Costar.name              count
----------------------   -----
Carrie-Anne Moss            3
Laurence Fishburne          3
Sandra Bullock              1
...
```

---

## What this query does

This query finds:

1. Keanu Reeves.
2. Movies he acted in.
3. Other actors in those same movies.
4. Counts how many movies they appeared in together.
5. Sorts the results.

---

### Relationship Pattern

```text
(Person)
     |
 ACTED_IN
     |
     V
(Movie)
     ^
     |
 ACTED_IN
     |
(Person)
```

This finds actors connected through the same movie.

---

### Breaking it down

```cypher
MATCH
```

Searches the graph.

---

```cypher
(Keanu:Person)
```

Finds a Person node.

Variable:

```text
Keanu
```

---

```cypher
(m:Movie)
```

Movie node.

---

```cypher
(Costar:Person)
```

Another Person node.

---

```cypher
WHERE
```

Filters:

```text
Keanu.name = "Keanu Reeves"
```

---

```cypher
DISTINCT
```

Removes duplicate co-stars.

Without it, someone appearing in multiple matching paths could be listed multiple times.

---

```cypher
count(*)
```

Counts how many shared movies they appeared in.

---

```cypher
ORDER BY
```

Sorts results.

First:

```text
Highest movie count
```

Then:

```text
Alphabetically by name
```

---

```cypher
DESC
```

Descending order.

Largest values first.

# Display the Graph Visually

```cypher
MATCH (Keanu:Person)-[:ACTED_IN]->(m:Movie)<-[r:ACTED_IN]-(Costar:Person)
WHERE Keanu.name = "Keanu Reeves"
RETURN Costar, m, Keanu, r;
```

### Result

In the Neo4j Browser, this displays an interactive graph showing Keanu Reeves, the movies, co-stars, and the `ACTED_IN` relationships between them.

---

## What this query does

Instead of returning individual properties (like names or titles), it returns the **entire nodes and relationships**. This allows the Neo4j Browser to render a visual graph.

# Degrees of Kevin Bacon

The **Six Degrees of Kevin Bacon** is the idea that almost every actor can be connected to Kevin Bacon through six or fewer co-acting relationships.

Graph databases are excellent at solving this kind of problem.

# Find the Shortest Path

```cypher
MATCH path = shortestPath(
    (Bacon:Person {name:"Kevin Bacon"})-[*]-(Keanu:Person {name:"Keanu Reeves"})
)
RETURN path;
```

### Result

In the Neo4j Browser, you'll see the shortest chain of actors and movies connecting Kevin Bacon and Keanu Reeves.

---

## What this query does

Finds the **shortest path** between two nodes by traversing their relationships.

---

### Breaking it down

```cypher
shortestPath()
```

Built-in function.

Searches for the shortest route between two nodes.

---

```cypher
path =
```

Stores the result in a variable called `path`.

---

```cypher
[*]
```

Means:

```text
Follow any relationship
Any number of times
```

The `*` indicates a variable-length path.

---

# Find Only the Distance

```cypher
MATCH path = shortestPath(
    (Bacon:Person {name:"Kevin Bacon"})-[*]-(Keanu:Person {name:"Keanu Reeves"})
)
RETURN length(path);
```

### Result

Example:

```text
length(path)
------------
2
```

---

## What this query does

Instead of returning the entire path, it returns only the number of relationships in that path.

---

### Breaking it down

```cypher
length(path)
```

Returns the number of relationships in the path.

This is useful when you only care about the degree of separation.

---

# Limit Path Length

```cypher
MATCH path = shortestPath(
    (Bacon:Person {name:"Kevin Bacon"})-[*..5]-(Keanu:Person {name:"Keanu Reeves"})
)
RETURN path;
```

### Result

Returns the shortest path only if it is within five relationships.

---

## What this query does

The pattern:

```cypher
[*..5]
```

means:

* Search any relationship type.
* Stop after at most **5 relationships**.

This prevents Neo4j from exploring extremely large graphs unnecessarily.

---

# Show the Path as Readable Text

```cypher
MATCH path = shortestPath(
    (First:Person {name:"Kevin Bacon"})-[*..5]-(Second:Person {name:"Keanu Reeves"})
)
UNWIND nodes(path) AS node
RETURN coalesce(node.name, node.title) AS text;
```

### Result

Example:

```text
Kevin Bacon
Movie A
Actor B
Movie B
Keanu Reeves
```

---

## What this query does

Instead of returning the whole graph object, it expands the path into individual nodes and displays either the person's name or the movie title.

---

### Breaking it down

```cypher
nodes(path)
```

Returns every node along the path as a list.

---

```cypher
UNWIND
```

Converts a list into individual rows.

For example:

Before:

```text
[
 Kevin Bacon,
 Movie,
 Actor,
 Movie,
 Keanu Reeves
]
```

After `UNWIND`:

```text
Kevin Bacon

Movie

Actor

Movie

Keanu Reeves
```

Each list element becomes its own row.

---

```cypher
AS node
```

Creates a variable named `node` for each element produced by `UNWIND`.

---

```cypher
coalesce(node.name, node.title)
```

`coalesce()` returns the **first non-null value**.

This is useful because:

* Person nodes have a `name`.
* Movie nodes have a `title`.

So:

```text
Person
→ name

Movie
→ title
```

Both can be displayed with one expression.

---

```cypher
AS text
```

Renames the output column to `text`.

# Find People Within Four Relationship Hops

```cypher
MATCH (Halle:Person)-[:ACTED_IN*1..4]-(Recommendation:Person)
WHERE Halle.name = "Halle Berry"
RETURN DISTINCT Recommendation.name
ORDER BY Recommendation.name;
```

### Result

Returns people connected to Halle Berry through one to four `ACTED_IN` relationships.

---

## What this query does

This query searches outward from Halle Berry through multiple `ACTED_IN` relationships to find a wider network of actors.

This type of query is commonly used in:

* Recommendation systems
* Social networks
* Network analysis

---

### Breaking it down

```cypher
[:ACTED_IN*1..4]
```

This means:

* Follow only `ACTED_IN` relationships.
* Travel at least **1** relationship.
* Travel at most **4** relationships.

The `*1..4` syntax defines the allowed path length.

---

```cypher
DISTINCT
```

Removes duplicate people from the results.

---

```cypher
ORDER BY Recommendation.name
```

Sorts the names alphabetically.

---

# Search Across Any Relationship

Instead of:

```cypher
[:ACTED_IN*1..4]
```

You can write:

```cypher
[*1..4]
```

This tells Neo4j to follow **any relationship type**, such as:

* `ACTED_IN`
* `DIRECTED`
* `WROTE`
* `PRODUCED`

This broadens the search and includes more kinds of connections.

---

# Key Points

* `MATCH (n)` finds all nodes in the graph.
* `labels()` returns the labels assigned to a node.
* `type()` returns the type of a relationship.
* `count(*)` counts matching rows, similar to SQL.
* `DISTINCT` removes duplicate results.
* `ORDER BY` sorts query results and `DESC` sorts in descending order.
* Returning whole nodes (`RETURN Costar, m, Keanu, r`) allows the Neo4j Browser to display an interactive graph.
* `shortestPath()` finds the minimum path between two nodes.
* `length(path)` returns the number of relationships in a path.
* `[*..5]` limits how far Neo4j searches, improving performance.
* `UNWIND` expands a list into individual rows.
* `nodes(path)` extracts all nodes from a path.
* `coalesce()` returns the first non-null value, making it useful when different node types store their display values in different properties.
* `[:ACTED_IN*1..4]` follows `ACTED_IN` relationships between one and four hops, while `[*1..4]` follows any relationship type.

# One-Sentence Summary

**Cypher enables powerful graph queries by allowing you to count nodes and relationships, traverse multiple levels of connected data, find the shortest paths between nodes, and analyze networks using concise, readable graph patterns.**
