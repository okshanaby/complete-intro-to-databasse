# Complex Neo4j Queries — Simple Summary

These notes introduce more advanced **Cypher** queries that help you analyze and explore data in a Neo4j graph database.

You'll learn how to:

* Count nodes by their labels.
* Count different relationship types.
* Find actors who worked together.
* Count how many movies actors appeared in together.
* Sort query results.
* Display results as an interactive graph in the Neo4j Browser.

Unlike earlier queries that simply created or found data, these queries focus on **analyzing relationships** within the graph.

---

# Prerequisite

These queries assume you have already inserted all the movie and actor data from the previous lesson.

---

# Count Nodes by Label

```cypher
MATCH (n)
RETURN DISTINCT labels(n), count(*);
```

### Result

Example output:

```text
labels(n)     count(*)
-----------   --------
["Person"]    8
["Movie"]     1
```

---

## What this query does

This query searches for **every node** in the database, groups them by their labels, and counts how many nodes belong to each label.

For example, it tells you how many:

* Person nodes
* Movie nodes
* Director nodes (if any)

exist in your graph.

---

### Breaking down the query

```cypher
MATCH (n)
```

Matches **all nodes** in the graph.

Notice there is **no label** after `n`.

This means:

```text
Every node regardless of its type
```

---

```cypher
labels(n)
```

`labels()` is a built-in Cypher function.

It returns the label(s) assigned to a node.

Example:

```text
Person
Movie
Actor
Director
```

A node can have multiple labels.

---

```cypher
count(*)
```

Counts the number of matching rows.

This works similarly to SQL:

```sql
COUNT(*)
```

---

```cypher
DISTINCT
```

Removes duplicate label groups before displaying the results.

---

# Count Relationship Types

```cypher
MATCH (n)-[r]->()
RETURN type(r), count(*);
```

### Result

Example:

```text
type(r)     count(*)
----------  --------
ACTED_IN         7
DIRECTED         1
```

---

## What this query does

Instead of counting nodes, this query counts the number of relationships of each type in the graph.

For example:

* How many `ACTED_IN` relationships exist?
* How many `DIRECTED` relationships exist?

---

### Breaking down the query

```cypher
MATCH (n)-[r]->()
```

Matches:

* Any starting node
* Any relationship
* Any ending node

The pattern:

```text
(node)-[relationship]->(node)
```

means:

```text
Find every relationship in the graph
```

---

```cypher
[r]
```

Represents a relationship variable.

Here it's named `r`, but it could be any valid variable name.

---

```cypher
type(r)
```

Returns the type of the relationship.

Example:

```text
ACTED_IN
DIRECTED
MANAGED
WORKS_FOR
```

---

```cypher
count(*)
```

Counts how many relationships exist for each relationship type.

# Find Keanu Reeves' Co-stars

```cypher
MATCH (Keanu:Person)-[:ACTED_IN]->(m:Movie)<-[:ACTED_IN]-(Costar:Person)
WHERE Keanu.name = "Keanu Reeves"
RETURN DISTINCT Costar.name, count(*)
ORDER BY count(*) DESC, Costar.name;
```

### Result

Example:

```text
Costar.name             count(*)
---------------------   --------
Carrie-Anne Moss            3
Laurence Fishburne          3
Sandra Bullock              1
...
```

---

## What this query does

This query finds every actor who has appeared in the same movie as **Keanu Reeves**.

It also counts how many movies each actor has appeared in together with Keanu.

Finally, it sorts the results so that actors who worked with him the most appear first.

---

### Visualizing the relationship

The query follows this graph pattern:

```text
(Person)
     |
 ACTED_IN
     |
     V
(Movie)
     ^
     |
 ACTED_IN
     |
(Person)
```

This means:

1. Start from Keanu Reeves.
2. Find movies he acted in.
3. Find other actors connected to those movies.

---

### Breaking down the query

```cypher
MATCH
```

Searches the graph for the specified pattern.

---

```cypher
(Keanu:Person)
```

Matches a node with:

Variable:

```text
Keanu
```

Label:

```text
Person
```

---

```cypher
[:ACTED_IN]
```

Matches only `ACTED_IN` relationships.

---

```cypher
(m:Movie)
```

Matches movie nodes.

---

```cypher
(Costar:Person)
```

Matches another person who acted in the same movie.

---

```cypher
WHERE
```

Filters the results.

Only the person named:

```text
Keanu Reeves
```

is used as the starting point.

---

```cypher
RETURN DISTINCT Costar.name
```

Returns each co-star only once.

Without `DISTINCT`, duplicate rows could appear if multiple matching paths exist.

---

```cypher
count(*)
```

Counts how many movies each co-star appeared in together with Keanu.

---

```cypher
ORDER BY
```

Sorts the results.

The query sorts using **two columns**.

First:

```cypher
count(*) DESC
```

This sorts by:

* Highest number of shared movies first.

---

Second:

```cypher
Costar.name
```

If two actors have appeared in the same number of movies with Keanu, they are sorted alphabetically by name.

---

```cypher
DESC
```

Means:

```text
Descending order
```

Largest values appear first.

---

# Display the Graph Visually

```cypher
MATCH (Keanu:Person)-[:ACTED_IN]->(m:Movie)<-[r:ACTED_IN]-(Costar:Person)
WHERE Keanu.name = "Keanu Reeves"
RETURN Costar, m, Keanu, r;
```

### Result

If you run this query in the **Neo4j Browser**, you'll see an interactive graph showing:

* Keanu Reeves
* His co-stars
* Movies
* `ACTED_IN` relationships connecting them

---

## What this query does

This query is almost identical to the previous one.

The difference is that it returns the **entire nodes and relationships** instead of individual fields.

Because Neo4j Browser receives complete graph objects, it can render them visually.

---

### Breaking down the query

```cypher
RETURN Costar, m, Keanu, r;
```

Returns:

* `Costar`

  * Person node

* `m`

  * Movie node

* `Keanu`

  * Person node

* `r`

  * Relationship

Since these are graph objects rather than simple text values, Neo4j Browser displays them as connected nodes and edges.

---

# Why Return Nodes Instead of Properties?

Compare these two queries:

Returns only text:

```cypher
RETURN Costar.name;
```

Result:

```text
Carrie-Anne Moss
Laurence Fishburne
Sandra Bullock
```

Returns complete graph objects:

```cypher
RETURN Costar, m, Keanu, r;
```

Result in Neo4j Browser:

```text
(Keanu Reeves)
      |
 ACTED_IN
      |
(The Matrix)
      |
 ACTED_IN
      |
(Carrie-Anne Moss)
```

Returning full nodes and relationships allows Neo4j Browser to generate an interactive graph visualization.

---

# Key Points

* `MATCH (n)` matches every node in the graph.
* `labels()` returns the label(s) assigned to a node.
* `type()` returns the type of a relationship.
* `count(*)` counts matching rows, similar to SQL.
* `DISTINCT` removes duplicate results.
* `ORDER BY` sorts results, and `DESC` sorts in descending order.
* Graph patterns like `(Person)-[:ACTED_IN]->(Movie)<-[:ACTED_IN]-(Person)` make it easy to find connected data.
* Returning node properties (such as `Costar.name`) gives text results.
* Returning complete nodes and relationships enables Neo4j Browser to display an interactive graph.

# One-Sentence Summary

**Cypher's graph pattern matching makes it easy to analyze connected data by counting nodes and relationships, finding co-stars through shared movies, sorting aggregated results, and visualizing entire graphs in the Neo4j Browser.**
