# NodeJS with Neo4j — Simple Summary

This section shows how to connect a **Node.js application** to a **Neo4j graph database**.

The main goal is to run a graph query from a backend server and return the result to a simple frontend.

The important database concepts are:

* Connecting Node.js to Neo4j using the **Neo4j driver**.
* Using the **Bolt protocol** to communicate with Neo4j.
* Creating database sessions and closing them properly.
* Running **parameterized Cypher queries** to prevent injection attacks.
* Reading Neo4j query results from returned **Record objects**.
* Using graph traversal queries like finding the shortest path between two people.

## `server.js`

```js
import express from "express";
import neo4j from "neo4j-driver";

const CONNECTION_STRING =
  process.env.NEO4J_CONNECTION_STRING || "bolt://localhost:7687";

const driver = neo4j.driver(CONNECTION_STRING);

async function init() {
  const app = express();

  app.get("/get", async (req, res) => {
    const session = driver.session();
    const result = await session.run(
      `
        MATCH path = shortestPath(
            (First:Person {name: $person1 })-[*]-(Second:Person {name: $person2 })
        )
        UNWIND nodes(path) as node
        RETURN coalesce(node.name, node.title) as text;
    `,
      {
        person1: req.query.person1,
        person2: req.query.person2,
      },
    );

    res.json({
      status: "ok",
      path: result.records.map((record) => record.get("text")),
    });

    await session.close();
  });

  app.use(express.static("./static"));
  const PORT = process.env.PORT || 3000;
  app.listen(PORT);
  console.log(`running on http://localhost:${PORT}`);
}
init();
```

# Neo4j Connection Setup

The important part is the backend connection.

```javascript
import express from "express";
import neo4j from "neo4j-driver";

const CONNECTION_STRING =
  process.env.NEO4J_CONNECTION_STRING || "bolt://localhost:7687";

const driver = neo4j.driver(CONNECTION_STRING);
```

### What this does

This creates a connection between the Node.js application and the Neo4j database.

### Database Explanation

Neo4j communicates using the **Bolt protocol**.

Bolt is Neo4j's own database communication protocol, similar to:

* PostgreSQL wire protocol.
* MongoDB wire protocol.

Instead of sending SQL over HTTP, applications communicate directly using the database protocol.

Example:

```text
Node.js Application
        |
        |
      Bolt
        |
        |
     Neo4j Database
```

---

```javascript
const CONNECTION_STRING =

```

```text
NEO4J_CONNECTION_STRING
```


# Creating a Neo4j Driver

```javascript
const driver = neo4j.driver(CONNECTION_STRING);
```

The driver manages communication with Neo4j.

Similar to PostgreSQL connection pools:

* It manages connections.
* It reuses connections.
* It creates sessions when queries need to run.

---

# Creating a Database Session

```javascript
const session = driver.session();
```

A session is a temporary connection used for database operations.

Example:

```text
Application
     |
     |
 Driver
     |
     |
 Session
     |
     |
 Neo4j Query
```

After finishing the query:

```javascript
await session.close();
```

must be called.

Why?

Because database connections are limited resources.

Closing sessions allows Neo4j to reuse connections instead of creating unnecessary new ones.

# Running a Cypher Query

The application sends this Cypher query:

```cypher
MATCH path = shortestPath(
    (First:Person {name: $person1 })-[*]-(Second:Person {name: $person2 })
)
UNWIND nodes(path) as node
RETURN coalesce(node.name, node.title) as text;
```

## What the query does

It finds the shortest relationship path between two people in the graph.

Example graph:

```text
Tom Hanks
    |
 acted in
    |
 Movie A
    |
 directed by
    |
 Steven Spielberg
```

The query can find the connection between:

```text
Tom Hanks
```

and

```text
Steven Spielberg
```

---

# Cypher Query Explanation

## MATCH

```cypher
MATCH path = shortestPath(...)
```

Searches the graph.

`shortestPath()` finds the smallest number of relationships between two nodes.

---

## Nodes

```cypher
(First:Person {name: $person1 })
```

Finds a node:

* Label: `Person`
* Property: `name`

Example:

```text
Person {
    name: "Tom Hanks"
}
```

---

## Relationships

```cypher
-[*]-
```

Means:

"Follow any relationship type."

The query does not care whether the relationship is:

```text
ACTED_IN
DIRECTED
PRODUCED
```

It follows all possible connections.

---

## Parameters

```cypher
{name: $person1}
```

`$person1` is a parameter.

The actual value is sent separately:

```javascript
{
  person1: req.query.person1,
  person2: req.query.person2,
}
```

Example:

```text
person1 = "Tom Hanks"
person2 = "Steven Spielberg"
```

---

# Why Parameterized Queries Matter

Bad approach:

```javascript
"SELECT * FROM users WHERE name = '" + username + "'"
```

or:

```javascript
MATCH (p:Person {name:" + userInput + "})
```

The user input becomes part of the query.

An attacker could inject malicious commands.

---

Parameterized query:

```cypher
{name: $person1}
```

The database treats the value only as data.

Benefits:

* Prevents injection attacks.
* Safer queries.
* Better query planning.

This works similarly to PostgreSQL prepared statements.

# Processing Neo4j Results

Neo4j returns:

```text
Record objects
```

Example:

```javascript
result.records.map((record) => record.get("text"))
```

The query result is a list:

```text
[
  Record,
  Record,
  Record
]
```

Each record contains returned fields.

`get()` extracts a field:

```javascript
record.get("text")
```

Example:

Before:

```text
Record {
   text: "Tom Hanks"
}
```

After:

```text
"Tom Hanks"
```

# Important Database Concepts Summary

## 1. Neo4j Driver

The driver allows applications to communicate with Neo4j.

Equivalent examples:

| Database   | Driver         |
| ---------- | -------------- |
| PostgreSQL | pg             |
| MongoDB    | mongodb driver |
| Neo4j      | neo4j-driver   |

---

## 2. Bolt Protocol

Bolt is Neo4j's database communication protocol.

Instead of:

```text
Application → HTTP → Database
```

Neo4j uses:

```text
Application → Bolt → Neo4j
```

It is optimized for graph operations.

---

## 3. Sessions

Sessions are temporary database connections.

Always:

1. Create session.
2. Run query.
3. Close session.

Example:

```text
Open session
      |
Run query
      |
Close session
```

---

## 4. Cypher

Cypher is Neo4j's query language.

It is similar to SQL but designed for graphs.

SQL:

```sql
SELECT *
FROM users;
```

Cypher:

```cypher
MATCH (person:Person)
RETURN person;
```

---

## 5. Parameterized Queries

Always pass values separately:

Good:

```cypher
{name:$person1}
```

Bad:

```text
Insert user input directly into query string
```

Advantages:

* Security.
* Prevents injection.
* Cleaner queries.

---

## 6. Neo4j Query Results

Neo4j returns:

```text
Records
```

You extract values using:

```javascript
record.get("fieldName")
```

Example:

```javascript
record.get("text")
```

returns the value of:

```cypher
RETURN ... AS text
```

---

# One-Sentence Summary

**Node.js connects to Neo4j using the Bolt protocol and the Neo4j driver, creates sessions to run safe parameterized Cypher queries, and processes returned Record objects to retrieve graph data such as shortest paths between nodes.**
