# 6 Degrees of Kevin Bacon Problem — Simple Summary

The **Six Degrees of Kevin Bacon** is a famous idea that says:

> **Almost every actor or actress can be connected to Kevin Bacon through six or fewer acting relationships.**

For example:

```text
Kevin Bacon
   │
Acted with
   │
Actor A
   │
Acted with
   │
Actor B
   │
Acted with
   │
Keanu Reeves
```

Each connection represents two actors appearing in the **same movie**.

This is an ideal problem for a **graph database** because graph databases are designed to efficiently find connections between nodes.

In Neo4j, you can easily find the shortest connection between two actors using the built-in `shortestPath()` function.

---

# Find the Shortest Path Between Kevin Bacon and Keanu Reeves

```cypher
MATCH path = shortestPath(
    (Bacon:Person {name:"Kevin Bacon"})-[*]-(Keanu:Person {name:"Keanu Reeves"})
)
RETURN path;
```

### Result

If you run this in the **Neo4j Browser**, you'll see an interactive graph showing the shortest path connecting Kevin Bacon and Keanu Reeves.

Example:

```text
Kevin Bacon
     │
 ACTED_IN
     │
A Few Good Men
     │
 ACTED_IN
     │
Kevin Pollak
     │
 ACTED_IN
     │
The Usual Suspects
     │
 ACTED_IN
     │
Keanu Reeves
```

(The exact path depends on the data stored in your database.)

---

## What this query does

This query searches for the **shortest chain of relationships** connecting two people.

Instead of manually searching through every actor and movie, Neo4j automatically explores the graph and returns the shortest path.

---

### Breaking down the query

```cypher
MATCH
```

Searches the graph for a matching pattern.

---

```cypher
path =
```

Stores the result in a variable named:

```text
path
```

The variable contains the **entire path**, including:

* All nodes
* All relationships

---

```cypher
shortestPath(...)
```

`shortestPath()` is a built-in Cypher function.

It finds the shortest route between two nodes by following their relationships.

This is one of the strengths of graph databases.

---

```cypher
(Bacon:Person {name:"Kevin Bacon"})
```

Starting node.

It matches a `Person` whose name is **Kevin Bacon**.

---

```cypher
(Keanu:Person {name:"Keanu Reeves"})
```

Ending node.

---

```cypher
[*]
```

This is a **variable-length relationship**.

It means:

* Follow **any relationship type**.
* Follow **any number of relationships**.

Neo4j will automatically determine the shortest route.

---

```cypher
RETURN path;
```

Returns the complete path so Neo4j Browser can display it as a graph.

# Return Only the Number of Connections

```cypher
MATCH path = shortestPath(
    (Bacon:Person {name:"Kevin Bacon"})-[*]-(Keanu:Person {name:"Keanu Reeves"})
)
RETURN length(path);
```

### Result

Example:

```text
length(path)
------------
4
```

---

## What this query does

Instead of returning the full graph, this query returns only the **number of relationships** in the shortest path.

This tells you the degree of separation between the two actors.

---

### Breaking down the query

```cypher
length(path)
```

Returns the number of relationships in the path.

For example:

```text
Kevin Bacon
    │
Movie
    │
Actor
    │
Movie
    │
Keanu Reeves
```

The length is the total number of relationships connecting the nodes.

---

# Limit the Search Distance

```cypher
MATCH path = shortestPath(
    (Bacon:Person {name:"Kevin Bacon"})-[*..5]-(Keanu:Person {name:"Keanu Reeves"})
)
RETURN path;
```

### Result

Returns the shortest path only if it can be found within **5 relationships**.

If no path exists within five relationships, no result is returned.

---

## What this query does

The pattern:

```cypher
[*..5]
```

limits how far Neo4j is allowed to search.

Without this limit:

```cypher
[*]
```

Neo4j may have to search a very large graph, which can take longer.

---

### Breaking down the query

```cypher
[*..5]
```

Means:

* Follow any relationship type.
* Traverse **up to 5 relationships**.
* Stop searching beyond that limit.

This improves query performance on large graphs.

# Display the Path as Readable Text

```cypher
MATCH path = shortestPath(
    (First:Person {name:"Kevin Bacon"})-[*..5]-(Second:Person {name:"Keanu Reeves"})
)
UNWIND nodes(path) AS node
RETURN coalesce(node.name, node.title) AS text;
```

### Result

Example output:

```text
text
-------------------
Kevin Bacon
Apollo 13
Bill Paxton
Twister
Keanu Reeves
```

Instead of displaying a graph, the query lists each node in the shortest path in order.

---

## What this query does

This query:

1. Finds the shortest path.
2. Extracts every node in that path.
3. Converts the list of nodes into individual rows.
4. Displays either the person's name or the movie title.

This format is much easier to read in **cypher-shell**.

---

### Breaking down the query

```cypher
nodes(path)
```

Returns all nodes contained in the path.

Example list:

```text
[
 Kevin Bacon,
 Apollo 13,
 Bill Paxton,
 Twister,
 Keanu Reeves
]
```

---

```cypher
UNWIND
```

`UNWIND` converts a list into separate rows.

Before:

```text
[
 Kevin Bacon,
 Apollo 13,
 Bill Paxton
]
```

After:

```text
Kevin Bacon

Apollo 13

Bill Paxton
```

Each list item becomes its own row.

---

```cypher
AS node
```

Stores each item from `UNWIND` in a variable named:

```text
node
```

---

```cypher
coalesce(node.name, node.title)
```

`coalesce()` returns the **first non-null value**.

This is useful because:

* Person nodes have a `name`.
* Movie nodes have a `title`.

For example:

Person:

```text
name = "Kevin Bacon"
title = null
```

Movie:

```text
name = null
title = "Apollo 13"
```

`coalesce()` automatically returns whichever property exists.

---

```cypher
AS text
```

Renames the output column to:

```text
text
```

# Find People Within Four Degrees of Halle Berry

```cypher
MATCH (Halle:Person)-[:ACTED_IN*1..4]-(Recommendation:Person)
WHERE Halle.name = "Halle Berry"
RETURN DISTINCT Recommendation.name
ORDER BY Recommendation.name;
```

### Result

Example:

```text
Recommendation.name
-------------------
Angela Bassett
Bill Paxton
Hugh Jackman
Patrick Stewart
...
```

(The actual results depend on your dataset.)

---

## What this query does

This query finds people connected to Halle Berry through **one to four `ACTED_IN` relationships**.

In other words, it explores a wider acting network around Halle Berry.

This type of query is commonly used in:

* Recommendation systems
* Social networks
* Network analysis
* Finding similar people or content

---

### Breaking down the query

```cypher
[:ACTED_IN*1..4]
```

This means:

* Follow only `ACTED_IN` relationships.
* Travel **at least 1 relationship**.
* Travel **at most 4 relationships**.

The `*1..4` syntax defines the minimum and maximum path length.

---

```cypher
WHERE Halle.name = "Halle Berry"
```

Starts the search from Halle Berry.

---

```cypher
RETURN DISTINCT Recommendation.name
```

Returns each recommended person only once.

`DISTINCT` removes duplicates.

---

```cypher
ORDER BY Recommendation.name;
```

Sorts the results alphabetically.

---

# Search Across All Relationship Types

Instead of searching only `ACTED_IN` relationships:

```cypher
[:ACTED_IN*1..4]
```

You can search **every relationship type**:

```cypher
[*1..4]
```

This tells Neo4j:

* Follow `ACTED_IN`
* Follow `DIRECTED`
* Follow `WROTE`
* Follow `PRODUCED`
* Follow any other relationship

This creates a much larger network because all connection types are considered.

---

# Key Points

* The **Six Degrees of Kevin Bacon** demonstrates how graph databases efficiently solve connection problems.
* `shortestPath()` finds the shortest route between two nodes.
* `[*]` means follow any relationship type for any number of hops.
* `[*..5]` limits the search to a maximum of five relationships, improving performance.
* `length(path)` returns the number of relationships in the path.
* `nodes(path)` extracts every node from a path.
* `UNWIND` converts a list into individual rows.
* `coalesce()` returns the first non-null value, making it useful when different node types use different properties.
* `[:ACTED_IN*1..4]` traverses between one and four `ACTED_IN` relationships.
* `[*1..4]` traverses one to four relationships of any type.
* These types of graph traversals are commonly used in recommendation engines, social networks, fraud detection, and knowledge graphs.

# One-Sentence Summary

**Neo4j makes it easy to solve network problems like the Six Degrees of Kevin Bacon by using graph traversal functions such as `shortestPath()`, allowing you to efficiently find connections, measure degrees of separation, and explore related nodes across multiple relationship levels.**
