# NodeJS with Redis — Simple Summary

**Redis with Node.js** allows applications to store and retrieve data extremely quickly using Redis as an in-memory database.

In this example, Redis is used for two common use cases:

1. **Counting data** — storing page views.
2. **Caching expensive operations** — saving results temporarily to avoid repeating slow database queries or API calls.

Redis is commonly used for:

* Caching
* Sessions
* Counters
* Rate limiting
* Queues
* Temporary data storage
* High-speed telemetry data

# Creating the Redis Backend

`server.js`

```javascript
import express from "express";
import redis from "redis";

const client = redis.createClient(); 
// defaults to localhost:6379, but you can also change that here

async function init() {
  await client.connect();

  const app = express();

  app.get("/pageview", async (req, res) => {
    const views = await client.incr("pageviews");

    res.json({
      status: "ok",
      views,
    });
  });

  const PORT = process.env.PORT || 3000;

  app.use(express.static("./static"));

  app.listen(PORT);

  console.log(`running on http://localhost:${PORT}`);
}

init();
```

# Redis Caching Example

Caching stores expensive results temporarily.

Instead of:

```
User request
      |
      v
Database query (slow)
      |
      v
Return result
```

Caching:

```
User request
      |
      v
Check Redis
      |
      +---- Exists → Return cached data
      |
      +---- Missing → Run expensive operation → Save result
```

---

Add this to `server.js`:

```javascript
// under the client = redis.createClient() line

function cache(key, ttl, slowFn) {
  return async function (...props) {
    const cachedResponse = await client.get(key);

    if (cachedResponse) {
      return cachedResponse;
    }

    const result = await slowFn(...props);

    await client.setEx(key, ttl, result);

    return result;
  };
}


async function verySlowAndExpensiveFunction() {

  console.log("oh no an expensive call!");

  const p = new Promise((resolve) => {

    setTimeout(() => {
      resolve(new Date().toUTCString());
    }, 5000);

  });

  return p;
}


const cachedFn = cache(
  "expensive_call",
  10,
  verySlowAndExpensiveFunction
);


// inside init, under app.get pageviews

app.get("/get", async (req, res) => {

  const data = await cachedFn();

  res.json({
    data,
    status: "ok",
  });

});
```

# Cache Function Explanation

## Function Definition

```javascript
function cache(key, ttl, slowFn)
```

Receives:

| Parameter | Meaning                           |
| --------- | --------------------------------- |
| `key`     | Redis key name                    |
| `ttl`     | How long data stays cached        |
| `slowFn`  | Function to run if cache is empty |

Example:

```javascript
cache(
 "expensive_call",
 10,
 verySlowAndExpensiveFunction
)
```

means:

```
Store result as "expensive_call"
Keep it for 10 seconds
Use this function to generate data
```

```javascript

```

Redis command:

```

Example:

```

```

Store:

```

```

```

# Thundering Herd Problem

The caching example has a possible problem.

Imagine:

* Cache expires.
* 1,000 users request the same data.

All requests see:

```text
Cache missing
```

Then all run:

```text
verySlowAndExpensiveFunction()
```

Result:

```text
1000 expensive database queries
```

This is called the:

**Thundering Herd Problem**

---

# Better Solution: Cache Locking

A common approach:

Create a second Redis key:

```text
expensive_call_lock
```

Flow:

```text
Request
 |
 | Check cache
 |
 ├── Exists
 |      |
 |      return data
 |
 └── Missing
        |
        Check lock
        |
        ├── Lock exists
        |       |
        |       wait
        |
        └── No lock
                |
                Create lock
                |
                Calculate data
                |
                Save cache
                |
                Remove lock
```

This prevents many requests from doing the same expensive work.


```
Request 1:
    Cache missing
    Create lock

Request 2:
    Cache missing
    See lock exists
    Wait

Request 1:
    Finish calculation
    Store result
    Remove lock

Request 2:
    Get cached result
```

The lock prevents many users from running the same expensive operation.

# Final Summary

Redis with Node.js is commonly used to make applications faster by storing frequently accessed or temporary data in memory.

The two main patterns shown are:

1. **Counters**

```javascript
client.incr("pageviews")
```

Used for:

* Views
* Likes
* Metrics

2. **Caching**

```javascript
client.get()
client.setEx()
```

Used for:

* Expensive database queries
* Slow APIs
* Computations

Redis reduces database load and improves application performance by avoiding unnecessary repeated work.
