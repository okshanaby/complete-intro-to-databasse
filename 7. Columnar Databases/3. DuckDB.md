# DuckDB — Simple Summary

**DuckDB** is an **embedded analytical (OLAP) database** designed for fast data analysis. Unlike PostgreSQL or MongoDB, DuckDB **does not run as a database server**. Instead, it works directly with a database file, similar to SQLite.

DuckDB is especially good for:

* Data analysis
* Reporting
* Exploring large datasets
* Reading Parquet, CSV, and JSON files
* Exporting results to files like CSV

It is **not intended** to replace PostgreSQL or MySQL for web applications.

---

# Key Points

* Embedded database (no server required)
* Uses SQL
* Optimized for analytical (OLAP) workloads
* Reads Parquet files directly
* Can query remote files (such as those stored in Amazon S3)
* Can export query results to CSV
* Stores data in a columnar format for fast analytics

---

# DuckDB vs Traditional Databases

Most databases you've used so far work like this:

```text
Application

↓

Database Client (psql, mongosh)

↓

Database Server

↓

Database Files
```

Examples:

* PostgreSQL
* MySQL
* MongoDB

The client sends queries to a running database server.

---

DuckDB works differently.

```text
DuckDB CLI

↓

Database File (.db)
```

There is **no database server**.

The DuckDB program directly opens and modifies the database file.

This is exactly how SQLite works.

---

# MotherDuck

**MotherDuck** is a cloud service built around DuckDB.

Instead of working only with local files, MotherDuck lets you:

* Store DuckDB databases in the cloud
* Share databases
* Query cloud data
* Work collaboratively

Think of it like:

* SQLite → Turso
* DuckDB → MotherDuck

The local DuckDB CLI still works directly with files.

# Querying a Remote Parquet File

```sql
SELECT * FROM read_parquet(
  's3://us-prd-motherduck-open-datasets/netflix/netflix_daily_top_10.parquet'
) LIMIT 10;
```

## Result

Returns the first 10 rows from a Parquet file stored in Amazon S3.

Example:

```text
Title                  Rank   Type
---------------------------------------
Wednesday              1      TV Show
Stranger Things        2      TV Show
The Gray Man           3      Movie
...
```

---

## Query Explanation

### What it does

Instead of importing the file first, DuckDB reads the Parquet file directly from cloud storage.

This means:

```text
S3 File

↓

DuckDB

↓

SQL Query

↓

Results
```

No manual download is required.

---

### `read_parquet()`

```sql
read_parquet(...)
```

Reads a Parquet file as if it were a database table.

The file can be:

* Local
* Network storage
* Amazon S3
* Other supported locations

---

### `LIMIT 10`

```sql
LIMIT 10
```

Returns only the first 10 rows.

Useful for previewing large datasets.

# Importing the Parquet File into DuckDB

```sql
CREATE TABLE netflix AS
SELECT *
FROM read_parquet(
  's3://us-prd-motherduck-open-datasets/netflix/netflix_daily_top_10.parquet'
);
```

## Result

Creates a new table:

```text
netflix
```

containing all rows from the Parquet file.

---

## Query Explanation

### What it does

Reads the Parquet file and stores its contents permanently in the local DuckDB database.

After this:

```text
Parquet File

↓

DuckDB Table

↓

SQL Queries
```

No need to read the remote file again.

---

### `CREATE TABLE`

Creates a database table.

---

### `AS SELECT`

Populates the table using the results of a query.

---

### `read_parquet()`

Loads the Parquet file into the table.

When calculating:

```text

```

This greatly improves analytical query performance.

# Finding Popular Shows During COVID

```sql
SELECT Title, Type, MAX("Days In Top 10") as peak_days
FROM netflix
WHERE "As of" BETWEEN '2020-03-15' AND '2020-06-30'
GROUP BY ALL
ORDER BY peak_days DESC
LIMIT 10;
```

## Result

Example:

```text
Title               Type       Peak Days
----------------------------------------
Tiger King          TV Show    60
Ozark               TV Show    57
...
```

---

## Query Explanation

### What it does

Finds the shows that stayed in Netflix's Top 10 the longest during the COVID lockdown period.

---

### `WHERE ... BETWEEN`

```sql
WHERE "As of"
BETWEEN '2020-03-15'
AND '2020-06-30'
```

Filters rows whose dates fall within the specified range.

---

### `MAX("Days In Top 10")`

Finds the highest value of **Days In Top 10** for each show.

---

### `GROUP BY ALL`

Groups by every non-aggregated column in the `SELECT` statement.

Equivalent to writing:

```sql
GROUP BY Title, Type
```

The advantage is convenience. If you add or remove non-aggregated columns in the `SELECT` list, DuckDB automatically updates the grouping without requiring you to edit the `GROUP BY` clause.

---

### `ORDER BY peak_days DESC`

Sorts from highest to lowest.

---

### `LIMIT 10`

Returns only the top 10 results.

---

# Finding the Best Netflix Exclusives

```sql
SELECT
  Title,
  Type,
  MAX("Days In Top 10") as total_days,
  ROUND(AVG("Viewership Score"), 1) as avg_score
FROM netflix
WHERE "Netflix Exclusive" = 'Yes'
GROUP BY ALL
ORDER BY total_days DESC
LIMIT 10;
```

## Result

Example:

```text
Title          Total Days    Avg Score
--------------------------------------
Wednesday      90            98.5
The Crown      85            96.3
...
```

---

## Query Explanation

### What it does

Analyzes Netflix-exclusive content to identify which shows remained popular the longest and achieved the highest average viewership scores. This type of analysis can help Netflix decide which kinds of exclusive content are worth investing in.

---

### `AVG()`

Calculates the average viewership score.

---

### `ROUND(..., 1)`

Rounds the average to one decimal place.

Example:

```text
98.4567

↓

98.5
```

# Exporting Results to CSV

```sql
COPY (
  SELECT Title, Type, MAX("Days In Top 10") as peak_days
  FROM netflix
  GROUP BY ALL
  ORDER BY peak_days DESC
) TO 'top_titles.csv' (HEADER);
```

## Result

Creates:

```text
top_titles.csv
```

Example:

```csv
Title,Type,Peak Days
Wednesday,TV Show,90
The Crown,TV Show,85
...
```

---

## Query Explanation

### What it does

Runs the query and saves its results directly into a CSV file instead of displaying them in the terminal.

This makes it easy to:

* Share results
* Import into Excel
* Create reports
* Build dashboards

---

### `COPY`

Exports query results to a file.

---

### `TO`

Specifies the destination file.

```text
top_titles.csv
```

---

### `(HEADER)`

Adds column names as the first row of the CSV file.

Without it:

```csv
Wednesday,TV Show,90
```

With it:

```csv
Title,Type,Peak Days
Wednesday,TV Show,90
```

# Why DuckDB Is Popular

DuckDB focuses on making analytical tasks simple and efficient.

Features highlighted in these notes include:

* Directly querying Parquet files
* Querying remote files without importing them first
* Easy aggregation with `GROUP BY ALL`
* Reading columnar data efficiently
* Exporting results directly to CSV
* Embedded architecture with no server setup

These features make DuckDB especially useful for data exploration, reporting, and analytics.
