# Delta Lake, Iceberg, and Parquet — Simple Summary

When working with **OLAP (analytical) databases**, **how data is stored on disk matters a lot** because it directly affects:

* Query speed
* Storage efficiency
* Reliability
* Concurrent access
* Data updates

Unlike databases such as PostgreSQL or MongoDB, where storage is mostly hidden from developers, data engineers often work directly with storage formats like **Parquet**, **Iceberg**, and **Delta Lake**.

---

# Key Points

* **Parquet** is the standard file format for analytical data.
* **Iceberg** adds database-like features on top of Parquet.
* **Delta Lake** is Databricks' version of a table format with similar goals.
* Most modern OLAP systems support Parquet.
* Many also support Iceberg and Delta Lake.
* DuckDB can read and write these formats, although some advanced features are still limited.

---

# From Data Lake to Data Warehouse

Originally, big data systems separated storage into two parts.

## Data Lake

A **Data Lake** stores raw, unprocessed data.

Examples:

* CSV files
* JSON files
* Parquet files
* Log files
* Images
* Sensor data

Visualization:

```text
Raw Data

CSV
JSON
Images
Logs

↓

Data Lake
```

Nothing needs to be cleaned before storing it.

---

## Data Warehouse

A **Data Warehouse** stores processed, cleaned, and organized data that is ready for analysis.

Visualization:

```text
Data Lake

↓

Clean Data

↓

Data Warehouse

↓

Analytics
```

The warehouse contains structured data optimized for reporting and business intelligence.

---

# Data Lakehouse

A **Data Lakehouse** combines the advantages of both a Data Lake and a Data Warehouse.

Instead of copying data between two systems, analytics can be performed directly on data stored in the lake.

Visualization:

```text
Raw Data

↓

Data Lakehouse

↓

Analytics
```

Benefits:

* One storage system
* Less data duplication
* Lower storage costs
* Simpler architecture
* Faster analytics

Databricks popularized this architecture.

# Apache Parquet

**Apache Parquet** is the most widely used storage format for analytical data.

It is a **columnar file format**, meaning it stores data by columns rather than rows.

Example table:

| ID | Name  | Age |
| -- | ----- | --- |
| 1  | Alice | 25  |
| 2  | Bob   | 31  |
| 3  | John  | 40  |

Internally, Parquet stores it like:

```text
ID
1
2
3

Name
Alice
Bob
John

Age
25
31
40
```

This makes analytical queries much faster because only the required columns are read.

---

# Why Parquet Is Popular

Advantages:

* Columnar storage
* Very fast analytical queries
* Excellent compression
* Small file sizes
* Supported by nearly every analytics platform

It has become the standard storage format for big data.

---

# Limitations of Parquet

Although Parquet is an excellent file format, it is **not a complete database**.

Some limitations include:

* No ACID transactions
* No version history
* Updating data often requires rewriting the file
* Limited support for concurrent writes

Think of Parquet as a highly optimized data file, not a full database system.

---

# ACID Compliance

Many databases guarantee **ACID** properties.

| Letter | Meaning     |
| ------ | ----------- |
| A      | Atomicity   |
| C      | Consistency |
| I      | Isolation   |
| D      | Durability  |

These properties ensure transactions remain safe and reliable, even if failures occur.

Parquet files alone do not provide these guarantees.

---

# Apache Iceberg

**Apache Iceberg** is a **table format** built on top of storage formats like Parquet.

Instead of replacing Parquet, Iceberg manages Parquet files and adds database-like capabilities.

Visualization:

```text
Applications

↓

Apache Iceberg

↓

Parquet Files
```

Iceberg provides features such as:

* ACID transactions
* Schema evolution (changing table structures)
* Time travel (querying previous versions)
* Better handling of concurrent writes
* Metadata management

Because of these features, Iceberg is widely supported by modern analytical databases.

---

# Delta Lake

**Delta Lake** is another table format created by **Databricks**.

Like Iceberg, it sits on top of Parquet files.

Visualization:

```text
Applications

↓

Delta Lake

↓

Parquet Files
```

Delta Lake also provides:

* ACID transactions
* Version history
* Schema evolution
* Reliable updates
* Concurrent write support

It was developed because Databricks wanted additional features tailored to its platform.

---

# Iceberg vs Delta Lake

Both aim to solve the same problem: making Parquet behave more like a database.

| Iceberg                | Delta Lake                         |
| ---------------------- | ---------------------------------- |
| Apache project         | Created by Databricks              |
| Built on Parquet       | Built on Parquet                   |
| Open standard          | Originally Databricks-focused      |
| Widely supported       | Strong integration with Databricks |
| Adds database features | Adds database features             |

For most users, both provide similar capabilities.

---

# Iceberg Version 3

The notes mention **Apache Iceberg v3**.

This version incorporates many improvements, including features that Databricks needed for its own ecosystem.

The goal is to reduce fragmentation by encouraging more systems to adopt a common open standard.

---

# DuckDB Support

DuckDB can work with all three formats:

| Format     | DuckDB Support                      |
| ---------- | ----------------------------------- |
| Parquet    | Full support                        |
| Iceberg    | Supported (some operations limited) |
| Delta Lake | Supported (some operations limited) |

Parquet currently has the most complete support in DuckDB.

---

# Relationship Between Them

A simple way to think about them is:

```text
Parquet

↓

Stores Data Efficiently

──────────────────────────

Iceberg

↓

Manages Parquet Files
Adds Database Features

──────────────────────────

Delta Lake

↓

Also Manages Parquet Files
Adds Database Features
```

Parquet is the storage format, while Iceberg and Delta Lake are management layers that make working with Parquet safer and more powerful.

---

# Why Data Engineers Care About These Formats

For analytical systems containing billions of rows, choosing the right storage format affects:

* Query performance
* Storage costs
* Data reliability
* Update efficiency
* Compatibility across analytics tools

That's why discussions around Parquet, Iceberg, and Delta Lake are common in data engineering.

---

# One-Sentence Summary

**Parquet is the standard columnar file format for analytical data, while Apache Iceberg and Delta Lake are table formats built on top of Parquet that add database features such as ACID transactions, schema evolution, versioning, and safer data management for modern data lakehouse architectures.**
